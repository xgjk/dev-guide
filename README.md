# dev-guide — AI 文档与业务知识库

本仓库集中存放 **AI 相关文档编写规范** 与 **各产品线的业务 / API 说明**，供 AI 辅助开发、产品口径统一及团队协作引用。

> **【全库强制】文档修订记录**  
> 本仓库内 **每一份 Markdown 文档**（含 `01.规范`、`02.产品业务AI文档`、`03.其它常用信息` 下全部 `.md`）均须在正文前部设有 `**## 修订记录`**，并使用**统一四列表格**：**版本**、**日期**、**变更摘要**、**变更人**。  
> **每次内容变更必须在表格末尾追加一行**，不得删改、覆盖或删除历史行，便于人与 AI 快速了解「何时、谁、改了什么」。行级差异以 Git 为准。  
> API / 业务说明类文档主要服务于 **编写与维护 Skill、对齐 Agent 口径**；修订记录有助于后续判断是否需要同步更新 Skill 包或 Agent 说明（具体联动规则可再行细化）。

## 使用建议

1. **写新业务说明**：先读 [业务说明文档编写规范](01.规范/业务说明文档编写规范.md)，再落笔。
2. **写或改 Open API 文档**：对照 [开放平台 API 编写规范](01.规范/开放平台API编写规范.md)；大文档拆分参见 [API 文档拆分与组织规范](01.规范/API文档拆分与组织规范.md)。
3. **定义新 Agent**：使用 [Agent 定义规范 V1](01.规范/Agent_定义规范_V1.md) 中的模板。
4. **实现或评审 Skill 包**：严格遵循 [skill 编写规范](01.规范/skill/skill编写规范.md) 中的目录、绑定关系与验证清单，并可用 [Skills 规范校验清单](01.规范/skill/Skills 规范校验清单.md) 做交付前检查；构建流程见 [Skill 包构建流程](01.规范/skill/Skill 包构建流程.md)。
5. **配置 OpenClaw / 钉钉 / 玄关 IM**：见上文 **03.其它常用信息** 表格；或直接浏览 [该目录](03.其它常用信息/) 下的 `.md` 文件。
