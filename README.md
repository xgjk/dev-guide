# dev-guide — AI 文档与业务知识库

本仓库集中存放 **AI 相关文档编写规范** 与 **各产品线的业务 / API 说明**，供 AI 辅助开发、产品口径统一及团队协作引用。

> **【全库强制】文档修订记录**  
> 本仓库内 **每一份 Markdown 文档**（含 `01.AI文档规范`、`02.产品业务AI文档` 下全部 `.md`）均须在正文前部设有 **`## 修订记录`**，并使用**统一四列表格**：**版本**、**日期**、**变更摘要**、**变更人**。  
> **每次内容变更必须在表格末尾追加一行**，不得删改、覆盖或删除历史行，便于人与 AI 快速了解「何时、谁、改了什么」。行级差异以 Git 为准。  
> API / 业务说明类文档主要服务于 **编写与维护 Skill、对齐 Agent 口径**；修订记录有助于后续判断是否需要同步更新 Skill 包或 Agent 说明（具体联动规则可再行细化）。

## 01.AI文档规范

面向 **AI 学习与人类编写** 的统一规范，新建或修订文档时建议先对照对应规范。

| 文档 | 用途概要 |
|------|----------|
| [业务说明文档编写规范](01.AI文档规范/业务说明文档编写规范.md) | 业务语义说明：系统是什么、核心对象与关系、规则与易混点；**不包含**接口调用与页面操作细节 |
| [开放平台 API 编写规范](01.AI文档规范/开放平台API编写规范.md) | Open API 文档结构、章节写法与接口模板（概述、通用说明、流程、接口明细等） |
| [Agent 定义规范 V1](01.AI文档规范/Agent_定义规范_V1.md) | 业务 Agent 的标准定义结构（定位、场景、职责边界、所需 Skill 能力等） |
| [Skill 编写规范（XGJK Skill 包协议）](01.AI文档规范/skill编写规范.md) | Skill 包目录结构、`SKILL.md` / OpenAPI 文档 / Python 脚本绑定与验证清单 |

规范之间的典型关系：**业务说明** 讲「是什么」→ **Agent 定义** 讲「助理做什么」→ **Skill / API 规范** 讲「能力如何落地与文档化」。

## 02.产品业务AI文档

按业务域存放 **业务说明**、**API 说明** 及 **Agent 规范文档**（含部分 Skill 拆分目录）。

当前覆盖示例如下（以仓库内实际文件为准）：

| 业务域 | 典型内容 |
|--------|----------|
| BP | BP 系统业务说明、API 说明；编制修订 / 审计 / 数字对齐 / 进展分析 / 评价等 Agent 规范 |
| 工作协同 | 业务说明、API 说明；汇报帮写、汇报分析、任务协同等 Agent 规范 |
| 基础服务 / 知识库 / AI慧记 / 新媒体 / SFE | 各域业务说明或开放平台使用说明、API 说明等 |

各产品线文档命名与章节结构，请同时遵循 `01.AI文档规范` 中的对应规范。

## 使用建议

1. **写新业务说明**：先读 [业务说明文档编写规范](01.AI文档规范/业务说明文档编写规范.md)，再落笔。
2. **写或改 Open API 文档**：对照 [开放平台 API 编写规范](01.AI文档规范/开放平台API编写规范.md)。
3. **定义新 Agent**：使用 [Agent 定义规范 V1](01.AI文档规范/Agent_定义规范_V1.md) 中的模板。
4. **实现或评审 Skill 包**：严格遵循 [skill 编写规范](01.AI文档规范/skill编写规范.md) 中的目录、绑定关系与验证清单。

## 版本与维护

- 各文档的版本与日期除「文档信息」等章节外，**必须以该文档内的「修订记录」表格为准**；每次修改务必追加一行。
- 本仓库为文档型仓库，无强制构建或安装步骤；使用 Markdown 阅读器或 IDE 即可浏览。

如有新增产品线或规范升级，建议在本 README 的对应表格中补充条目，便于检索。
