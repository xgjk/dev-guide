## 二、核心规则（共 10 条，全部可衡量）

### 规则 1：返回字段 ≤ 完成当前任务所需

- **禁止**默认返回以下类型字段：头像 URL、图标 URL、颜色值、CSS 类名、纯装饰文案、前端渲染专用字段。
- 每个接口的响应参数表必须用一列标注 `AI决策`（是/否），标注为"否"的字段默认不返回，需调用方显式请求才返回。

### 规则 2：嵌套层级 ≤ 3 层

- 响应 JSON 的 `data` 到叶子字段，最多 3 层嵌套（即 `data → 对象 → 字段`）。
- 超过 3 层时，必须拆为**列表/摘要接口**（返回 ≤ 3 层）+ **详情接口**（按需返回更深结构），并在接口说明中标注"此字段仅在详情接口返回"。

### 规则 3：数据量必须标注上限

每个接口必须明确以下数字（写在"基本信息"表格中）：

| 指标 | 说明 |
|---|---|
| 默认返回条数 | 分页接口的默认 pageSize（如 20） |
| 最大返回条数 | 单次请求最多返回多少条（如 200） |
| 单字段最大字符数 | 文本类字段的最大长度（如 500 字） |
| 预估响应体积 | 典型场景下的 JSON 大小（如"约 2KB / 20条"） |

### 规则 4：相同含义 → 相同命名

| 含义 | 全文档统一命名 | 禁止出现的变体 |
|---|---|---|
| 主键 | `id` | `Id`、`ID`、`objId`、`objectId` |
| 名称 | `name` | `displayName`、`showName`、`title`（除非业务含义确实不同） |
| 状态 | `status` | `state`、`statusCode` |
| 创建时间 | `createTime` | `createdAt`、`createdTime`、`gmtCreate` |
| 更新时间 | `updateTime` | `updatedAt`、`modifiedTime`、`gmtModified` |

新增字段时，先查本文档"五、公共数据结构"中是否已有同义字段，有则复用命名。

### 规则 5：相似接口必须对比

当同一模块存在功能相近的多个接口（如"按部门查询"与"按人员查询"），必须：

1. **放在相邻章节**（如 4.1、4.2）。
2. 在接口说明开头增加"**对比说明**"小节，格式：

```
接口 A（本接口）与接口 B 的区别：
- 入参差异：A 接收 deptId，B 接收 userId
- 返回差异：A 返回部门维度汇总，B 返回个人维度明细
- 业务场景：A 用于部门报告，B 用于个人任务查看
- 选择建议：需要部门级数据时用 A，需要个人明细时用 B
```

### 规则 6：去重——同一事实只返回一次

- 若列表中每条记录都包含相同的父级字段（如所有 task 都有相同的 `projectName`），该字段必须提升到列表上层，每条记录中删除。
- 若父对象已返回 `parentId` + `parentName`，子对象中不得重复返回，除非子对象脱离父上下文后仍需独立使用——此时必须在字段说明中写明原因。

### 规则 7：空值表达统一

全文档统一：

| 场景 | 约定 |
|---|---|
| 无数据 / 不适用 | `null` |
| 空文本 | `""`（空字符串） |
| 空列表 | `[]`（空数组） |

禁止出现 `null`、`""`、`"-"`、`"暂无"`、`"无"` 混用表达同一语义。若接口无法遵守，必须在字段说明中写明例外。

### 规则 8：长文本必须截断或摘要（列表接口）

长文本规则**仅适用于列表 / 摘要类接口**，详情接口不受此约束。

**列表接口：**

- 单个文本字段超过 **1000 字**时，默认返回截断摘要（≤ 200 字）+ 原文标识（如 `hasFullContent: true`），供调用方判断是否需要进一步请求详情接口获取全文。
- 富文本（HTML、带格式文本）**禁止**直接返回原文，更禁止返回包含 `base64` 图片的内容。统一转为纯文本或 Markdown。
- 表格数据保留为 Markdown 表格或结构化数组，禁止压平为纯文本段落。

**详情接口：**

- **不限制**单字段文本长度，应返回字段的完整内容。
- 富文本与表格数据的格式要求同上（纯文本/Markdown，禁止 base64 图片）。

### 规则 9：渐进式加载——每个接口标注"数据量级别"

每个接口在"基本信息"表格中必须包含一行：

```
数据量级别：小（≤ 1KB）/ 中（1-10KB）/ 大（> 10KB，需分页）
```

AI 可据此决定：
- 小：直接调用，无需特殊处理
- 中：单次可用，注意 token 消耗
- 大：必须分页，必须传 pageSize + pageNum/cursor

### 规则 10：接口定义格式——固定模板，不多不少

每个接口必须且只能包含以下小节（无内容的小节写"无"）：

| 小节 | 必填 |
|---|---|
| 一句话说明 | ✅ |
| 基本信息（含数据量级别） | ✅ |
| 请求参数表 | ✅ |
| 请求示例（curl） | ✅ |
| 响应参数表（含 AI决策 列） | ✅ |
| 响应示例（JSON） | ✅ |
| 对比说明（仅相似接口） | 相似接口时必填 |
| 数据流向（返回值如何被其他接口使用） | 推荐 |
| 数据口径（统计/状态等特殊含义） | 有则必填 |
| 权限与安全 | 与通用说明不同时必填 |

## 三、通用说明（每个文档必写的固定内容）

### 3.1 访问地址

```
https://{域名}/open-api/{接口地址}
```

### 3.2 环境信息

| 环境 | Base URL | 备注 |
|---|---|---|
| 生产 | `https://xxx.example.com` | - |

### 3.3 公共请求头

| Header | 必填 | 说明 |
|---|---|---|
| `appKey` | 是 | 应用密钥 |
| `Content-Type` | POST/PUT 必填 | `application/json` |

### 3.4 通用响应结构

```json
{
  "resultCode": 1,
  "resultMsg": null,
  "data": null
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `resultCode` | Integer | `1` = 成功，其他 = 失败 |
| `resultMsg` | String | 成功时 `null`，失败时错误描述 |
| `data` | T | 业务数据，失败时 `null` |

### 3.5 认证

所有接口通过请求头 `appKey` 鉴权，仅应用级。`appKey` 申请流程不在本文档范围。

### 3.6 错误码

| resultCode | 说明 | AI 处理动作 |
|---|---|---|
| 1 | 成功 | 读取 data |
| 0 | 通用失败 | 读取 resultMsg 展示给用户 |
| 401 | 鉴权失败 | 停止重试，提示用户检查 appKey |
| 429 | 限流 | 等待 5 秒后重试，最多 3 次 |
| 500 | 系统异常 | 稍后重试，最多 3 次 |

```markdown

{一句话说明用途及在流程中的角色。}

**基本信息**

| 项目 | 说明 |
|---|---|
| 接口地址 | `/{path}` |
| 请求方式 | GET / POST / PUT / DELETE |
| Content-Type | application/json（POST/PUT 时） |
| 数据量级别 | 小 / 中 / 大 |
| 默认返回条数 | {N} |
| 最大返回条数 | {N} |
| 预估响应体积 | {约 X KB / N 条} |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `param` | String | 是 | {枚举值/格式/取值范围} |

**请求示例**

```bash
curl -X GET 'https://{域名}/open-api/{path}?param=xxx' \
  -H 'appKey: XXXXXXXX'
```

**响应参数**

`data` 类型为 `{Type}`：

| 字段名 | 类型 | AI决策 | 说明 |
|---|---|---|---|
| `field1` | Long | 是 | {说明} |

**响应示例**

{JSON 示例}

**对比说明**（仅相似接口）

{与相邻接口的差异，见规则 5}

**数据流向**

- 返回的 `{field}` 用作 **4.x {接口名}** 的 `{param}` 入参。

**数据口径**（有统计/状态特殊含义时填写）

| 项 | 说明 |
|---|---|
| 时间口径 | {时区 + 自然日/账期} |

**权限与安全**（与 3.5 不同时填写）

## 六、变更管理

| 项 | 约定 |
|---|---|
| 新增字段 | 必须可选（有默认值），不得破坏现有 AI Skill 解析 |
| 新增枚举值 | 允许，AI 必须能处理未知枚举值（跳过或展示原文） |
| 删除字段 | 先标注废弃 ≥ 2 周，再下线 |
| `null` vs 字段缺失 | 全文档等价，AI 按 `null` 处理即可 |
| 数组顺序 | 无特殊说明则无语义 |
| 修改记录 | 每次变更必须在文档顶部修订记录表中登记 |
