## 第二章 使用入口与端区分（必读）

AI慧记 存在**三个相互独立的用户入口**，数据在账号下统一汇总，但**入口形态与部分能力不同**，请先确认您使用的是哪一种。

| 入口名称 | 端类型 | 典型使用环境 | 说明 |
|----------|--------|--------------|------|
| **钉钉工作中的「AI慧记」** | **WEB 端** | 浏览器（PC 或手机浏览器访问钉钉工作台） | 在钉钉工作台内打开，以网页形式使用，适合会议文件上传、纪要整理与集中管理。 |
| **手机工作说说中的「AI慧记」** | **移动端 App** | 手机「工作说说」应用内 | 在手机端使用，支持线下会议录音、拍照标记、声纹等，充分利用手机硬件能力。 |
| **电脑视讯通客户端中的入口「AI慧记」** | **桌面客户端内嵌** | 电脑已安装的「视讯通」客户端 | 在视频会议场景中，从视讯通会议界面进入，用于会中/会后自动生成纪要并同步到账号。 |

### 2.1 钉钉工作中的「AI慧记」——WEB 端

- **入口**：钉钉 → 工作台 → AI慧记（网页打开）。
- **形态**：浏览器内运行的 Web 应用。
- **主要能力**：会议记录列表与详情查看、上传音视频文件转写、会议摘要/原文/智能提问、**会议报告（智能报告/图文报告）**、待办任务与协同分发、分享与回收站、热词配置、个人知识库沉淀等**偏管理与消费**的能力。
- **典型场景**：上传已有录音/录像做纪要、在电脑上集中查看多场会议、做问答与分享。

### 2.2 手机工作说说中的「AI慧记」——移动端

- **入口**：手机打开「工作说说」App → 进入 AI慧记。
- **形态**：移动端应用（非浏览器）。
- **主要能力**：**会中录制**（实时录音、拍照、标记）、声纹录入与识别、动态摘要、边录边问、后台录音/悬浮窗等**偏采集与实时交互**的能力；也支持查看「我的记录」、摘要与简单问答。
- **典型场景**：线下会议录音、面试/需求讨论现场记录、外出时快速查看某场会议要点。

### 2.3 电脑视讯通客户端中的入口「AI慧记」——视讯通端

- **入口**：在电脑上打开「视讯通」客户端 → 会议中或会议后通过客户端内入口进入 AI慧记 能力。
- **形态**：桌面客户端内嵌的入口，依赖视讯通会议能力。
- **主要能力**：与**当前视频会议**绑定，实现会中/会后**自动录制并生成会议记录**，纪要自动同步至 AI慧记 账号；在客户端内可查看本场纪要，完整管理仍在 WEB 端进行。
- **典型场景**：线上视频会议自动出纪要，无需会后手动上传；需保留会议视频时，可通过账号/手机号匹配自动同步会议视频至 AI慧记。

### 2.4 三端关系小结

- **数据统一**：三个入口登录同一账号后，会议记录统一在「我的记录」中展示与管理。
- **能力侧重**：  
  - **采集与实时** → 优先用 **手机工作说说** 或 **视讯通**；  
  - **上传、整理、管理、深度问答与分享** → 优先用 **钉钉 WEB 端**。

## 第三章 产品价值与产品概述

### 3.1 产品价值说明（2026 版）

AI慧记 不限于“会议录音或转写”，而是帮助企业与个人**系统性降低会议成本、沉淀可复用业务知识，并保障会议数据安全可控**。

#### 3.1.1 面向企业与管理层

- **会议数据归属清晰、安全可控**：企业级统一采集与存储，录音、转写与纪要归属企业，降低分散与外泄风险。
- **过程与结果可追溯**：会议记录、分享记录与回收站机制，满足合规、审计与内部管理需求。
- **组织知识持续沉淀**：关键结论与决策背景以纪要形式保留，减少人员变动带来的知识流失。
- **降低管理沟通成本**：通过摘要与关键信息快速了解业务进展，无需参与所有会议。

#### 3.1.2 面向业务用户

- **减少记录与整理成本**：自动转写与摘要，无需会中分心记录、会后反复回听。
- **快速获取会议重点**：自动摘要与标记，短时间内掌握核心结论与讨论重点。
- **会议内容可提问与复用**：针对会议内容自然语言提问，无需人工翻记录。
- **统一管理多场会议**：线下、线上、外部文件均可在同一账号下集中管理。

### 3.2 产品定位

**AI慧记** 是一款面向企业用户的智能会议记录与会议内容理解工具，通过自动采集会议内容，将语音与视频转化为可阅读的文字记录、可快速理解的会议摘要、可直接提问与复用的会议知识。

### 3.3 典型解决问题

- 会中难以完整记录关键信息  
- 会后纪要整理耗时长、易遗漏  
- 会议内容难以回溯与复用  
- 会议记录分散，缺乏统一管理

## 第四章 会议记录的创建方式（按端说明）

### 4.1 实时录音创建会议记录

| 能力 | 手机工作说说（移动端） | 视讯通客户端 | 钉钉 WEB 端 |
|------|------------------------|--------------|-------------|
| 会中录音/录制 | ✓ 支持 | ✓ 会议内自动录制 | -|
| 会中拍照/截图 | ✓ 拍照，声画对齐 | 视讯通能力 | -|
| 会中标记 | ✓ | ✓ 截图、标记、拆分子纪要 | ✓|
| 动态摘要（如每 3 分钟） | ✓ | ✓ | ✓ |
| 边录边问 | - | - | ✓ |
| 子纪要拆分 | — | ✓ 会中按时间段拆分 | WEB 端可查看/管理 |

**说明**：实时录音与会中交互主要在**移动端**和**视讯通端**完成；WEB 端主要用于查看、编辑与管理。

### 4.2 上传音频或视频文件

- **适用端**：主要为 **钉钉工作中的 AI慧记（WEB 端）**。
- **支持格式**：mp3、mp4、wav 等常见格式。
- **基本流程**：进入 AI慧记 首页 → 选择「上传音视频」→ 选择本地或会议录制文件 → 提交并等待转写完成。

### 4.3 视频会议自动生成纪要

- **适用端**：**电脑视讯通客户端**（由企业配置 AI慧记 权限）。
- **流程**：在视讯通会议中开启 AI慧记录制 → 会议结束后，纪要在客户端内可查看，并**自动同步至 AI慧记 账号**，在 WEB 端/移动端「我的记录」中统一管理。
- **扩展**：需保留会议视频时，可通过账号与手机号匹配，自动同步企业会议视频至 AI慧记，无需手动下载或上传。

## 第五章 我的记录：会议内容统一管理

### 5.1 功能说明

「我的记录」在 **WEB 端与移动端** 均可用，用于集中展示该账号下的全部会议记录，支持：

- 按会议来源分类查看  
- 按会议名称搜索  
- 不同视图方式切换  

### 5.2 常用操作（各端可用范围以实际产品为准）

- 查看会议详情  
- 重命名会议记录  
- 分享会议内容  
- 删除会议记录（进入回收站）  

*批量管理、状态看板等进阶能力主要在 **WEB 端** 提供。*

## 第六章 会议详情与 AI 能力

### 6.1 会议摘要

系统自动生成会议摘要，便于快速把握核心内容与主要结论，**各端**进入会议详情后均可查看。

### 6.2 会议原文

会议原文为完整转写内容，按时间顺序展示。用户可：

- 播放对应音频或视频  
- 修改发言人名称  
- 查找关键词并批量修改  
- 筛选与过滤无效信息  

*完整编辑与批量操作以 **WEB 端** 为主。*

### 6.3 智能提问（AI 问答）

用户可针对会议内容自然语言提问，例如：

- 会议的主要结论是什么  
- 是否存在明确的行动事项  
- 某问题的讨论结果  
- 快速输出发言人观点  
- 基于模板快速生成内容  

**深度思考、指定标记段落定向提问、AI 改写与模板报告**等能力以 **WEB 端** 为主；移动端支持基础问答与摘要查看。

### 6.4 标记功能

- 会中标记：在**移动端**或**视讯通端**录制时添加。  
- 会后标记：可在详情页补充。  
- 用途：快速定位重点、按标记范围做更精确的总结与问答。

### 6.5 会议报告（智能报告 / 图文报告）

会议结束后，当会议内容**转写完成**，系统可**自动化生成会议报告**，支持多形态输出：

- **智能报告**：以结构化方式输出会议结论、讨论要点、风险与决策等（适合管理阅读与归档）。
- **图文报告**：以图文并茂的方式呈现关键信息（适合传播与汇报）。

报告支持以下操作（以 **WEB 端** 为主，其他端以实际产品为准）：

- **修改**：对报告内容进行编辑完善。
- **复制**：一键复制报告内容用于周报/邮件/IM 发送。
- **重新生成**：基于同一会议记录重新生成报告（用于换模板或调整侧重点）。
- **添加到个人知识库**：将高价值报告沉淀为可检索复用的个人知识资产。
- **导出图片**：图文报告支持导出为图片，便于分享与留存。

### 6.6 待办任务拆分与工作协同任务

系统可基于会议内容**自动拆分待办任务**（To-do），并将任务列表与会议结论关联，便于跟进闭环。

- **待办任务生成**：会议报告生成时同步输出待办项，或在会后从会议内容中抽取待办。
- **待办任务编辑**：支持补充责任人、截止时间、描述等关键信息（以 **WEB 端** 为主）。
- **生成工作协同任务**：待办任务支持一键生成「工作协同任务」，用于团队协作工具中分派与跟踪（具体对接范围以企业配置为准）。

## 第七章 分享与协作

### 7.1 会议分享

用户可将会议记录分享给同事查看，支持查看分享记录及查看人员情况。**以 WEB 端操作为主**，移动端可支持查看与简单分享（以实际产品为准）。

### 7.2 分享记录管理

在分享中心可查看：自己发出的分享、他人分享给自己的会议记录。**主要在 WEB 端**使用。

## 第八章 回收站与数据保护

- 删除的会议记录进入**回收站**，可恢复或彻底删除。  
- **各端**在「我的记录」或设置中均可进入回收站（具体以产品为准），机制用于防止误删、保障数据可控。

## 第九章 热词与声纹

### 9.1 热词功能

- 系统级企业与行业热词（集团产品、业务与常用术语）。  
- 用户可维护**个人常用热词**。  
- **配置入口**以 **WEB 端**为主，提升转写准确率。

### 9.2 声纹功能

- **录入**：主要在**手机工作说说（移动端）**完成个人声纹录制。  
- **效果**：转写时自动匹配发言人身份，支持多参会人声纹识别。  
- **场景**：面试、多人讨论等需区分「谁在说话」时使用。
