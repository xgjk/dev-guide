## 二、系统概述

### 系统定义

AI情报系统是一套面向研究分析、售前咨询、行业运营等内容生产场景的智能报告平台，在当前文档范围内，核心支撑“模版管理（查询/新建/编辑/发布/下架/删除）-生成报告-查看进度与结果-修改章节内容”的业务闭环。

### 解决的核心问题

1. **报告起稿成本高**：行业分析、竞品研究、市场洞察等内容通常需要人工搭结构、搜资料、逐段撰写，整体成本高。
2. **模版维护成本高**：当现有模版不满足场景时，缺少统一的新建、编辑、删除能力会导致沉淀效率低。
3. **选模版效率低**：用户在开始写报告前，难以快速找到适合当前任务的模版并理解其结构是否匹配。
4. **任务状态不透明**：报告生成是异步过程，用户往往不知道任务是否完成、卡在哪一步、下一步该做什么。
5. **历史报告定位难**：用户想回看过去报告或确认某次任务结果时，缺少统一的查询和定位方式。
6. **局部修改成本大**：用户只对某个章节不满意时，如果不能局部修改，就容易被迫重新处理整份报告。

### 在整体业务中的定位

AI情报系统在当前场景中定位于**模版管理 + 报告生成与修订助手**：

- **向前承接**：承接用户的报告目标、模版管理诉求和上下文信息。
- **向后输出**：输出报告任务状态、报告全文、章节内容和修改后的结果。
- **横向协同**：让用户可以在一个连续使用链路中完成模版查看/新建/编辑/发布/下架/删除、任务查询和章节修改，而不必分散处理。

### 不负责解决的问题

- **最终业务结论判断**：系统负责生成与整理报告内容，不替代用户做最终业务结论。
- **复杂模板编排能力**：本文档范围不包含模板协同编辑、模板审批流、多版本并行发布等复杂治理场景。
- **整份报告重跑与复杂修订编排**：本文档范围不展开单章节 AI 重跑、整份报告重合成等扩展处理流程。
- **外部事实绝对校验**：关键事实和结论仍需业务人员复核。

## 三、核心业务对象

### 3.1 报告模版

**定义**：报告模版是报告结构的标准化定义，用于说明一份报告应包含哪些章节和子章节。

**作用**：

- 作为发起报告任务的前置选择对象。
- 帮助用户快速判断某种报告结构是否适合当前业务目标。
- 降低重复搭建报告结构的成本。

**与其他对象的关系**：

- 报告任务基于报告模版发起。
- 报告模版决定报告章节的大体结构。

**常见误解**：

- 误解：模版就是已经生成好的报告。纠正：模版只提供结构，不等于已有具体内容。
- 误解：看到模版列表就等于理解模版。纠正：用户通常还需要查看模版详情，才能判断是否适合当前任务。

### 3.2 报告任务

**定义**：报告任务是一次具体的报告生成过程，表示系统基于某个模版和输入信息生成一份报告。

**作用**：

- 承载报告生成这次具体业务动作。
- 提供任务状态，让用户判断是否完成、是否还需等待。
- 作为后续查看报告详情的入口。

**与其他对象的关系**：

- 报告任务基于某个报告模版创建。
- 报告任务完成后产生报告章节内容。

**常见误解**：

- 误解：创建任务就代表已经拿到报告。纠正：任务只是生成过程的起点，报告内容需要在任务完成后查看。
- 误解：任务状态就等于内容质量。纠正：任务完成只表示流程完成，不代表内容一定已经满足业务预期。

### 3.3 报告章节

**定义**：报告章节是报告中的内容单元，包括章节及其子章节，是用户查看和修改报告的直接对象。

**作用**：

- 承载具体的报告内容。
- 支持用户按章节阅读，而不是一次性处理整份报告。
- 支持用户只修改有问题的章节内容。

**与其他对象的关系**：

- 报告章节从属于某个报告任务。
- 报告章节的结构来源于报告模版。
- 报告章节修改后可产生新的结果版本。

**常见误解**：

- 误解：章节只是展示层内容。纠正：章节是报告查看和直接编辑的核心业务单元。
- 误解：改一个章节就必须重做整份报告。纠正：当前场景支持只修改目标章节，不影响整份报告的其他部分。

### 3.4 结果版本

**定义**：结果版本是某个报告章节在生成后、被人工修改后形成的历史结果记录。

**作用**：

- 保留章节内容的修改过程。
- 帮助用户理解当前内容是如何变化而来的。
- 让章节修改具备可追溯性。

**与其他对象的关系**：

- 结果版本从属于具体的报告章节。
- 报告章节被直接编辑后，会形成新的结果版本。

**常见误解**：

- 误解：直接编辑后旧内容就没有了。纠正：系统可以保留章节的历史结果版本。
- 误解：版本就是不同报告。纠正：版本描述的是同一章节内容的变化，不是新的报告任务。

## 四、核心业务关系

### 业务主链路

```text
模版查询/新建/编辑/发布/下架/删除 → 发起报告任务 → 查询任务进度 → 查看报告详情 → 修改目标章节
```

1. **模版查询与理解**：用户先搜索模版并查看详情，判断该模版是否适合当前报告目标。
2. **模版新建/编辑/发布/下架/删除**：当现有模版不满足需求时，用户可新建、编辑、发布/下架或删除模版。
3. **发起报告任务**：用户基于选定模版和上下文信息创建报告生成任务。
4. **查询任务进度**：任务创建后，用户查询当前状态，确认是否完成。
5. **查看报告详情**：任务完成后，用户查看报告全文、章节结构和子章节内容。
6. **修改目标章节**：如果某个子章节不满意，用户直接覆盖该章节内容，而不是重做整份报告。

### 对象之间的关系

```text
报告模版
   │ 决定结构
   ▼
报告任务
   │ 生成结果
   ▼
报告章节
   │ 修改后形成
   ▼
结果版本
```

### 关键流转逻辑

#### 报告生成流转

```text
选定模版 → 创建任务 → 生成中 → 已完成 → 查看详情
```

#### 报告查询流转

```text
输入任务标识或筛选条件 → 查询报告列表或状态 → 定位目标报告 → 查看报告详情
```

#### 章节修改流转

```text
查看报告章节 → 定位目标子章节 → 直接覆盖新内容 → 保存为新结果版本
```

## 五、业务规则

### 5.1 显性规则

1. **模版前置规则**：报告任务必须基于一个明确的报告模版发起。
2. **模版状态切换规则**：模版通过 `state` 控制发布状态，`1` 为已发布（上架），`0` 为未发布（下架）。
3. **模版删除权限规则**：删除模版仅允许创建者和管理员操作。
4. **任务异步规则**：报告生成是一个任务过程，创建任务后需要查询状态，不能默认即时完成。
5. **详情后置规则**：只有定位到具体报告任务后，用户才能查看对应的报告详情和章节内容。
6. **章节定点修改规则**：用户修改报告时，直接面向具体章节或子章节进行，不以整份报告为唯一修改单位。
7. **版本留痕规则**：章节内容被直接编辑后，应形成可追溯的结果版本记录。

### 5.2 隐性规则

1. **先理解后生成规则**：用户在发起任务前，通常需要先理解模版结构是否符合当前目标。
2. **先定位后查看规则**：用户查询历史报告时，通常先通过列表或状态查询定位目标，再进入详情查看。
3. **局部修改优先规则**：当问题只在某个章节时，优先直接修改该章节，而不是重新处理全部内容。
4. **结果需人工确认规则**：系统生成或保存后的内容是否满足业务要求，仍需用户自行判断。

### 5.3 权限或范围限制

1. **模版访问限制**：用户只能查看和使用自己有权限访问的模版。
2. **模版维护限制**：用户只能新建、编辑、发布/下架、删除自己有权限操作的模版。
3. **报告访问限制**：用户只能查询和查看自己有权限访问的报告任务和报告内容。
4. **章节修改限制**：用户只能修改自己有权限操作的报告章节。
5. **结果使用限制**：系统生成的报告内容如需正式对外使用，仍应遵循企业既有审核要求。

### 5.4 特殊说明或例外

1. **模版列表与模版详情区别**：模版列表用于初步筛选，模版详情用于确认结构是否合适。
2. **任务状态与报告详情区别**：任务状态回答“报告是否生成完成”，报告详情回答“报告具体写了什么”。
3. **章节修改范围限制**：当前场景强调直接覆盖目标章节内容，不展开更复杂的 AI 多轮修订流程。
4. **版本查看例外**：版本查询关注的是章节修改历史，不是新的报告列表。
5. **删除操作例外**：删除模版前需确认目标 `mobanId` 且具备对应权限（创建者或管理员）。

## 六、常见歧义与边界

### 6.1 容易混淆的概念

| 歧义点 | 澄清说明 |
|--------|----------|
| 报告模版 vs 报告任务 | 模版定义结构，任务承载一次实际生成过程。 |
| 任务状态 vs 报告详情 | 任务状态说明流程进度，报告详情说明内容结果。 |
| 报告列表 vs 报告详情 | 报告列表用于定位目标报告，报告详情用于查看具体内容。 |
| 报告章节 vs 结果版本 | 报告章节是当前内容单元，结果版本是该内容单元的历史记录。 |

### 6.2 容易误解的规则

| 歧义点 | 澄清说明 |
|--------|----------|
| 发起任务后是否立刻有结果 | 不一定，报告生成是异步过程，需要查询状态。 |
| 查到报告列表是否等于拿到全文 | 不等于，列表只用于定位目标，全文需进入详情查看。 |
| 修改一个章节是否影响整份报告 | 当前场景中，章节直接编辑只针对目标章节，不要求重做整份报告。 |
| 版本记录是否等于新的报告 | 不等于，版本记录是同一章节内容的变化历史。 |

### 6.3 理解边界

| 边界点 | 澄清说明 |
|--------|----------|
| 本文档范围 | 覆盖模版浏览与理解、模版新建/编辑/发布/下架/删除、报告生成、报告查询、章节直接编辑。 |
| 不包含的扩展场景 | 不包含模板协同审批、多版本发布治理、复杂 AI 修订、整份报告重跑等扩展流程。 |
| 内容生成与业务判断 | 系统负责生成和展示内容，不替代用户做最终业务判断。 |
| 内容结果与事实校验 | 报告可作为业务参考，但关键事实和结论仍需人工复核。 |
