## 一、概述

场景管理是 TBS 训战后台的训练内容资产管理能力。一个场景不是简单的标题和剧本，而是由训练目标、业务域、科室、品种/产品、AI 角色画像、学员简报、AI 对象上下文、教练要求、知识、标签、热词、评分规则、提示词快照和素材生成状态共同组成的训练内容单元。

本文以外部系统和 AI 助手可调用为目标，重点描述自由对话场景的搜索、选择、详情、创建、更新、发布/下线、删除、素材生成和内容导出能力。PPT 演讲场景归属同一场景资产体系，但创建和更新走 `/pptscene` 独立接口，本文仅提供相关入口概览。

本版暂不对外提供以下能力的接口明细：剧本生成草稿、复制场景、恢复已删场景。

### 1.1 外部调用推荐能力

| 能力 | 推荐接口 | 说明 |
|---|---|---|
| 搜索场景 | `POST /scene/menu-search-scenes` | 默认搜索入口，应用 `scene:view` 权限和场景菜单数据范围 |
| 选择场景 | `POST /scene/scene-for-select` | 活动、通知、仪表盘等场景下拉候选 |
| 读取场景详情 | `POST /scene/getSceneDetail?id={id}` | 获取完整内容，用于编辑、预览、生成调用上下文 |
| 创建自由对话场景 | `POST /scene/createScene` | 新建自由对话场景，`sceneType` 固定为 `dialogue` |
| 更新自由对话场景 | `POST /scene/updateScene?id={id}` | 编辑字段、发布、下线、替换画像/知识/标签/热词 |
| 删除场景 | `POST /scene/deleteScene?id={id}` | 已发布场景不可直接删除，需先下线 |
| 生成素材 | `POST /scene/generateBriefingAudio`、`POST /scene/generateBackgroundImages`、`POST /scene/getImageTaskStatus` | 生成简报语音、背景图并查询任务状态 |
| 导出/读取聚合内容 | `POST /scene/getSceneYaml`、`POST /scene/getBestPractice`、`POST /scene/getProductKnowledge` | 给外部调用方读取场景上下文、演练要求和知识文本 |

## 二、通用说明

### 2.1 访问地址

详见 `_通用约定.md`：

```text
https://{域名}/open-api/tbs-admin{接口地址}
```

### 2.2 公共请求头和响应

| 项 | 说明 |
|---|---|
| 鉴权 | 请求头传 `appKey` |
| 请求体 | `POST` 接口默认使用 `application/json` |
| 响应结构 | `{ "resultCode": 1, "resultMsg": "success", "data": ... }` |
| 命名规范 | 请求体和 Query 参数中的业务字段使用 `camelCase` |
| ID 精度 | 后端 ID 为 `Long`，外部系统和 JS 建议统一按字符串读取和传递 |

### 2.3 权限和数据范围

| 接口 | 建议 |
|---|---|
| `/scene/menu-search-scenes` | 推荐作为外部搜索入口，校验场景菜单查看权限，并应用场景菜单数据范围 |
| `/scene/scene-for-select` | 推荐作为资源选择入口，按 `menuCode` 校验对应菜单权限和数据范围 |
| `/scene/searchScenes` | 历史/共享搜索接口，不承载菜单权限与菜单数据范围，不建议作为默认搜索入口 |
| `/scene/listScenes` | 兼容旧调用的列表接口，不建议作为管理页或默认入口 |

`menuCode` 用于告诉后端当前资源选择发生在哪个菜单上下文中，从而应用对应菜单的查看权限和数据范围。场景管理的创建、编辑、搜索和产品选择场景下，固定推荐传 `scene`。只有在其他业务菜单内选择资源时才传对应值：活动配置传 `activity`，仪表盘传 `dashboard`，通知规则传 `notification`，权限配置传 `auth`。

### 2.4 依赖基础资源

创建或更新场景时，通常需要先通过基础数据接口确认可用资源 ID。前置资源获取和权限口径详见 `04-基础数据管理.md` 的“场景创建前置资源 ID 获取”。

| 资源 | 推荐接口 | 用途 |
|---|---|---|
| 品种/产品 | `POST /drug/product-for-select` | 获取 `drugId`，并以返回的 `businessDomainId` 作为场景业务域主来源 |
| 业务域 | `POST /businessDomain/listBusinessDomains` | 仅在需要展示业务域列表、或产品未返回业务域时补充查询 |
| 科室 | `POST /department/listDepartments` | 获取 `departmentId`；科室不作为业务域反推来源，选择时应与产品业务域兼容或可不传 |
| 疾病 | `GET /disease/forResourceSelect` | 获取 `diseaseId`，当前编辑页不是主展示字段 |
| AI 角色画像 | `GET /rolePersona/forResourceSelect` | 获取复用画像的 `personaId` |
| 产品知识 | `GET /knowledge/forResourceSelect` | 获取 `knowledgeIds` |
| 标签 | `GET /tag/forResourceSelect` | 获取 `tagIds` |

## 三、关键业务流程说明

### 3.1 搜索并选择场景

外部调用方需要查找场景时，优先调用 `/scene/menu-search-scenes`。如果是在活动、通知规则、仪表盘等具体菜单下做资源选择，优先调用 `/scene/scene-for-select`，并传入对应 `menuCode`；如果是在场景管理自身上下文选择资源，传 `scene`。

典型流程：

1. 使用 `search`、`statuses`、`businessDomainIds`、`drugIds`、`sceneType` 等条件查找场景。
2. 从列表中取 `id`。
3. 调用 `/scene/getSceneDetail?id={id}` 获取完整内容。
4. 如果要作为活动或通知的候选项，使用 `/scene/scene-for-select` 获取轻量候选结构。

### 3.2 createScene 与 updateScene 在前端页面中的使用方式

前端不是两个独立页面，而是同一个自由对话场景编辑器根据是否已有场景 ID 自动分流。

| 页面动作 | 前端状态 | 实际接口 |
|---|---|---|
| 场景列表点击“新建场景”并选择“自由对话模式” | `initialScene = null`，没有 `dbId` | 首次保存调用 `POST /scene/createScene` |
| 创建成功后继续停留在编辑器 | 后端返回新场景 ID，前端立即调用详情接口并写入 `editorInitialData` | 后续保存改为 `POST /scene/updateScene?id={id}` |
| 场景列表点击“编辑” | 先调用详情接口加载完整场景，带入 `editorInitialData` | 保存调用 `POST /scene/updateScene?id={id}` |
| 训练调试页右侧快速编辑简报/上下文 | 已有 `selectedSceneDbId` | 局部调用 `POST /scene/updateScene?id={id}` |

所以，页面上的按钮只是保存动作入口，接口语义由“有没有场景 ID”决定。`createScene` 用于第一次落库，`updateScene` 用于已有场景的所有后续编辑、发布、下线和内容替换。

编辑器底部按钮和 `status` 的关系如下：

| 按钮 | 传入 `status` | 说明 |
|---|---:|---|
| 保存草稿 | 新建或草稿为 `1`；已发布保持 `2`；已下线保持 `0` | 保存当前版本，不一定改变状态 |
| 发布上线 | `2` | 草稿发布 |
| 保存发布版本 | `2` | 已发布场景保存后继续保持发布 |
| 下线场景 | `0` | 已发布场景下线 |
| 转为草稿 | `1` | 已下线场景转回草稿 |
| 恢复上线 | `2` | 已下线场景重新发布 |

发布前，前端会校验标题、业务域、品种/产品、代表简报、AI 对象上下文、学员演练要求；如果配置了自定义评分规则，评分总分必须等于 100。后端还会校验自由对话场景至少有一个 AI 角色画像，且当前实现仅支持一个 AI 角色画像。

### 3.3 创建或更新自由对话场景

结构化创建时，建议外部系统按以下顺序组装字段：

1. 先通过产品选择接口确认 `drugId`，并使用产品返回的 `businessDomainId` 作为场景业务域。
2. 按需选择 `departmentId`、`location`；不要用科室反推业务域，若科室归属与产品业务域冲突，后端会拒绝保存。
3. 写入 `repBriefing`，作为学员进入训练前看到的代表简报。
4. 写入 `doctorOnlyContext`，作为 AI 对象在对话中掌握的情境和状态。
5. 写入 `coachOnlyContext`，作为教练侧对学员演练目标、评分关注点和复盘要求的描述。
6. 写入 `personas`，至少一个；如复用已有画像，使用 `mode=reuse`。
7. 可选写入 `knowledgeIds`、`tagIds`、`hotwordContents`、`reviewConfig`。
8. `status=1` 保存草稿，`status=2` 发布，`status=0` 下线。

### 3.4 发布、下线和删除

发布和下线不使用独立接口，而是调用 `/scene/updateScene?id={id}` 修改 `status`。

| 操作 | 调用方式 | 关键规则 |
|---|---|---|
| 发布 | `updateScene`，请求体 `{ "status": 2 }`，必要时同时带完整内容 | 发布时必须有 AI 角色画像；评分规则如自定义，总分必须等于 100 |
| 下线 | `updateScene`，请求体 `{ "status": 0 }` | 下线后可再转草稿或恢复上线 |
| 删除 | `deleteScene?id={id}` | 已发布场景不能删除；草稿删除会清理关联数据；已下线场景进入删除态 |

### 3.5 素材生成

代表简报语音和背景图属于自由对话场景的素材能力。

| 素材 | 接口 | 触发条件 |
|---|---|---|
| 简报语音 | `/scene/generateBriefingAudio?id={id}` | 场景存在且 `repBriefing` 非空，TTS 服务已配置 |
| 背景图 | `/scene/generateBackgroundImages?id={id}` | 场景存在，图片服务已配置，场景关联的品种/产品名称非空 |
| 图片任务状态 | `/scene/getImageTaskStatus?sceneId={id}&taskId={taskId}` | 使用背景图接口返回的 `taskId` 查询 |

更新 `repBriefing` 会重置语音素材状态；更新标题、地点、科室、品种/产品、业务域或 `repBriefing` 会重置背景图素材状态。

## 四、核心枚举和数据模型

### 4.1 场景状态

| 值 | 名称 | 说明 |
|---:|---|---|
| `0` | 已下线 | 不能作为新的训练入口，通常可编辑或重新上线 |
| `1` | 草稿 | 未发布，可继续编辑 |
| `2` | 已发布 | 可被训练、活动等模块使用 |

### 4.2 场景类型

| 值 | 说明 |
|---|---|
| `dialogue` | 自由对话模式，使用 `/scene/createScene` 和 `/scene/updateScene` |
| `ppt` | PPT 演讲模式，创建和更新使用 `/pptscene` 接口 |

### 4.3 开场设置

| 值 | 说明 |
|---|---|
| `learner` | 学员先开场，默认值 |
| `ai` | AI 先开场 |

### 4.4 AI 角色画像输入模式

| 值 | 说明 |
|---|---|
| `reuse` | 复用全局可见的角色画像，需要传 `personaId` |
| `scene_private` | 创建场景私有画像，需要传 `personaDraft` |

### 4.5 知识建议状态

| 字段 | 类型 | 说明 |
|---|---|---|
| `hidden` | Boolean | 是否不再展示建议面板 |
| `collapsed` | Boolean | 是否收起建议面板 |
| `items` | Array | 建议项列表 |

`items` 元素：

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | String | 建议项 ID |
| `title` | String | 建议标题 |
| `description` | String | 建议说明 |
| `status` | String | `auto-linked`、`suggested`、`create` |
| `confidence` | Number | 置信度 |
| `matchedKnowledgeId` | String | 命中的知识 ID，建议按字符串处理 |
| `matchedKnowledgeTitle` | String | 命中的知识标题 |
| `actionStatus` | String | `pending`、`accepted`、`dismissed` |

## 五、接口详细说明

### 5.1 场景菜单分页搜索

| 项目 | 说明 |
|---|---|
| 接口地址 | `/scene/menu-search-scenes` |
| 请求方式 | POST |
| Content-Type | `application/json` |
| 推荐用途 | 外部默认搜索入口、场景管理页列表 |
| 响应数据 | `Page<SceneListItem>` |

**请求体参数**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `pageIndex` | Integer | 否 | 页码，从 1 开始，默认 `1` |
| `pageSize` | Integer | 否 | 每页条数，默认 `20` |

| `businessDomainIds` | Array<String> | 否 | 业务域 ID 列表 |

| `departmentIds` | Array<String> | 否 | 科室 ID 列表 |

| `excludeScenesLinkedToOtherActivities` | Boolean | 否 | 是否排除已被其他活动关联的场景 |
| `currentActivityId` | String | 否 | 编辑活动时传入，配合排除规则保留当前活动已选场景 |

## 七、错误与边界说明

| 场景 | 可能错误 | 调用方处理建议 |
|---|---|---|
| 发布场景但无 AI 画像 | `发布场景前请先配置 AI 角色画像` 或 `自由对话场景至少需要一个 AI 角色画像` | 先选择或创建 `personas` |
| 自由对话传多个 AI 画像 | `自由对话场景仅支持一个 AI 角色画像` | 保留一个默认画像 |
| 复用画像但未传 ID | `复用画像时必须选择已有画像` | 补充 `personaId` |
| 私有画像但无草稿 | `场景私有画像缺少画像草稿` | 补充 `personaDraft` |
| 发布时评分总分不是 100 | `发布场景时评分维度总分必须等于 100` | 调整 `reviewConfig.dimensions` |
| 评分维度重复或为空 | `评分维度名称不能为空/不能重复` | 去重并补充维度名称 |
| 产品与科室业务域冲突 | `部门/科室与品种/产品归属的业务领域不一致` | 以产品返回的业务域为准，重新选择兼容科室或不传科室 |
| 删除已发布场景 | `已发布的场景不允许删除，请先下线。` | 先调用 `updateScene` 将 `status` 改为 `0` |
| 生成语音但简报为空 | `Scene has no rep_briefing content` | 先补充 `repBriefing` |
| 生成背景图但产品为空 | `场景关联的品种/产品名称为空` | 先补充 `drugId` 或基础资料中的产品名称 |
| 素材服务未配置 | `TTS 服务未配置` 或 `图片服务配置未设置` | 提示管理员检查外部服务配置 |
