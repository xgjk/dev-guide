## 场景创建前置资源 ID 获取

创建或更新自由对话场景时，需要先准备产品、业务域、角色画像、知识、标签等资源 ID。推荐以**产品/品种**作为业务域的主来源：先选择产品，使用产品返回的 `businessDomainId` 写入场景；科室只作为可选业务属性，不用于反推业务域。

下面的列表是场景创建前置资源的**推荐调用顺序**，不是同一类权限接口清单。其中产品选择接口带菜单上下文和数据范围控制；场景菜单搜索、场景选择接口的权限口径见 `02-场景管理.md`。业务域、科室等基础列表主要用于展示、名称回填或后台维护，不建议承担外部选择时的权限过滤职责。

### 推荐调用顺序

| 顺序 | 目标 | 推荐接口 | 取值 | 权限口径 |
|---:|---|---|---|---|
| 1 | 选择产品/品种 | `POST /drug/product-for-select` | `id` 作为 `drugId`，`businessDomainId` 作为场景 `businessDomainId` | 场景管理固定传 `menuCode=scene`，按场景菜单查看权限和产品数据范围过滤 |
| 2 | 补充业务域展示 | `POST /businessDomain/listBusinessDomains` | `id`、`name` | 企业内基础列表，不按菜单数据范围过滤 |
| 3 | 选择科室 | `POST /department/listDepartments` | `id` 作为 `departmentId` | 企业内基础列表；调用方可按产品 `businessDomainId` 自行筛选兼容科室 |
| 4 | 选择复用画像 | `GET /rolePersona/forResourceSelect` | `id` 作为 `personas[].personaId` | 企业内资源下拉；可用 `sourceType` 过滤 |
| 5 | 选择产品知识 | `GET /knowledge/forResourceSelect?drugId={drugId}` | `id` 作为 `knowledgeIds[]` | 企业内资源下拉；建议按产品过滤 |
| 6 | 选择标签 | `GET /tag/forResourceSelect` | `id` 作为 `tagIds[]` | 企业内资源下拉；可用 `category` 过滤 |
| 7 | 选择疾病（可选） | `GET /disease/forResourceSelect?departmentId={departmentId}` | `id` 作为 `diseaseId` | 企业内资源下拉；当前场景编辑页不是主字段 |

### 业务域判定规则

| 规则 | 说明 |
|---|---|
| 产品优先 | 场景创建时，优先使用产品返回的 `businessDomainId` 作为 `businessDomainId` |
| 科室不反推业务域 | 不建议通过科室决定场景业务域；如果传入科室，需确保它与产品业务域兼容 |
| 后端会兜底校验 | 当产品和科室归属业务域不一致时，后端会返回 `部门/科室与品种/产品归属的业务领域不一致` |
| 业务域列表用途 | `/businessDomain/listBusinessDomains` 主要用于展示、名称回填或产品未返回业务域时人工确认 |

### 权限建议

不是所有基础列表接口都带同样的数据范围控制。涉及“用户在某个菜单上下文下能看到哪些产品/场景”的场合，应优先使用带 `menuCode` 的选择接口；普通基础列表可以作为展示、回填或管理端维护入口。

| 接口 | 是否按菜单权限/数据范围过滤 | 建议 |
|---|---|---|
| `POST /drug/product-for-select` | 是，依赖 `menuCode` | 场景创建/编辑固定传 `scene`；活动、通知、仪表盘等页面按所在菜单传对应值 |
| `POST /drug/listDrugs` | 否，企业内基础列表 | 基础数据管理页维护产品时使用 |
| `POST /businessDomain/listBusinessDomains` | 否，企业内基础列表 | 用于展示和补充名称，不作为权限过滤入口 |
| `POST /department/listDepartments` | 否，企业内基础列表 | 用于展示和补充科室，不作为权限过滤入口 |
| `GET /rolePersona/forResourceSelect` | 否，企业内资源下拉 | 场景选择复用画像时使用 |
| `GET /knowledge/forResourceSelect` | 否，企业内资源下拉 | 场景关联知识时使用，建议传 `drugId` |
| `GET /tag/forResourceSelect` | 否，企业内资源下拉 | 场景关联标签时使用 |
| `GET /disease/forResourceSelect` | 否，企业内资源下拉 | 场景补充疾病时使用 |

### 产品选择接口

| 项目 | 说明 |
|---|---|
| 接口地址 | `/drug/product-for-select` |
| 请求方式 | POST |
| Content-Type | `application/json` |
| 推荐用途 | 创建场景前选择产品，并反推出业务域 |

**请求体参数**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `menuCode` | String | 是 | 菜单上下文。场景管理创建/编辑固定推荐 `scene`；活动配置传 `activity`；仪表盘传 `dashboard`；通知规则传 `notification`；权限配置传 `auth` |
| `search` | String | 否 | 产品名称关键词 |
| `limit` | Integer | 否 | 返回上限，默认 `500`，最大 `1000` |

**请求示例**

```json
{
  "menuCode": "scene",
  "search": "缓释片",
  "limit": 50
}
```

**成功响应示例**

```json
{
  "resultCode": 1,
  "resultMsg": "success",
  "data": [
    {
      "id": "2001",
      "name": "XX缓释片",
      "businessDomainId": "18",
      "businessDomainName": "心血管"
    }
  ]
}
```

调用方应将 `id` 写入场景的 `drugId`，将 `businessDomainId` 写入场景的 `businessDomainId`。
