## 2. 系统概述

### 2.1 子域一句话定义

**陪练与考试**是 CMS智汇中基于**题库**的两类智能体能力：陪练侧重在对话形态下进行**随机或序列化**的一问一答巩固，并可呈现过程性得分；考试侧重在相同交互形态下按**考核规则**完成题目集合并形成**成绩与通关判定**。

### 2.2 子域解决的核心问题

- **拜访前/碎片时间记不住**：用短对话快速过知识点与话术要点。
- **只会背不会答**：用语音或文字的一问一答逼出即时反应，而非只看材料。
- **需要轻量考核**：用考试类智能体设定题量与达标线，形成可查询结果。
- **能力维度要对齐培训口径**：通过题目类型与维度类配置，把「考了什么能力」从直觉变成可统计口径（具体维度集合以企业配置为准）。

### 2.3 在 CMS智汇与 TBS 组合中的定位

- **CMS智汇（本子域）**：偏「基础题 / 快问快答」式的掌握度巩固与轻考核，适合碎片化与访前准备。
- **TBS**：偏「应用题 / 情境演练」式的沉浸式训战。  
两者可同时存在：先用 CMS 把知识点打牢，再用 TBS 做情境应用，不在本文展开 TBS 细节。

### 2.4 本子域不负责解决的问题

- 不替代真实拜访中的合规与医学终审责任。
- 不承担完整线下培训组织与签到排课等 LMS 全链路能力。
- 不在本文约定判题算法、模型版本、接口参数等技术实现。

## 3. 核心业务对象

### 3.1 题库

- **定义**：结构化题目与答案素材的集合，是陪练与考试的共同内容来源。
- **作用**：决定「问什么、标准答什么、如何归类到能力维度」。
- **与其他对象的关系**：被陪练类、考试类智能体引用；与题型、分值系数等配置共同影响结果解释。
- **常见误解**：题库更新不会自动等同于所有引用它的智能体已完成业务复核；发布与生效范围以企业流程为准。

### 3.2 陪练（智能体类型）

- **定义**：以对话为主要形态、基于题库进行练习巩固的智能体类型；强调过程参与与反复练习。
- **作用**：让代表在短会话中快速过题，形成「听见题—组织答—得到反馈」的闭环。
- **与其他对象的关系**：依赖题库；产生会话记录；可出现过程性得分展示（以配置为准）。
- **常见误解**：陪练不是正式考核制度本身；不等同考试通关记录。

### 3.3 考试（智能体类型）

- **定义**：以对话为主要形态、基于题库按考核规则完成题目集合的智能体类型；强调结果与达标。
- **作用**：在可控题量与评分维度下完成测验，沉淀成绩与历史记录供管理侧查询与导出。
- **与其他对象的关系**：依赖题库；关联成绩查询能力；与调研类在「采集 vs 评定」目标上不同。
- **常见误解**：考试得分高不等于所有业务场景合规无风险；需结合维度与制度理解。

### 3.4 题目与题型（业务语义）

- **定义**：题库中的最小考察单元；题型用于区分考察方式与评分权重口径。
- **作用**：把「考了什么」拆成可配置、可统计的结构。
- **与其他对象的关系**：被陪练/考试智能体抽取与呈现；与能力维度标签共同支撑培训口径对齐（标签体系以企业配置为准）。
- **常见误解**：题型名称不等于能力维度；二者可能有关联但不是同一概念。

### 3.5 成绩与考试历史（考试类）

- **定义**：考试类智能体在管理侧可查询的结果数据，包括分数、历史记录与维度类明细等。
- **作用**：支撑培训管理者抽查、对比与导出。
- **与其他对象的关系**：依附于考试智能体与用户；与会话记录并存但口径不同。
- **常见误解**：导出的是数据副本，不等于制度上的考核定案。

### 3.6 会话过程（陪练与考试共用）

- **定义**：用户通过文本、语音、图片等输入与系统交互的过程记录。
- **作用**：支撑复盘、分享、反馈与问题排查。
- **与其他对象的关系**：陪练与考试均可能产生会话；考试侧额外强调成绩沉淀。
- **常见误解**：会话里「像聊天」不等于没有考核规则；考试仍以题库与规则为主轴。

## 4. 核心业务关系

### 4.1 业务主链路（陪练）

1. 企业维护题库与能力维度口径。
2. 配置并发布陪练类智能体（绑定题库与练习策略）。
3. 用户进入陪练，系统按策略抽取题目并以对话推进。
4. 用户作答并获得过程反馈与（如有）得分展示。
5. 会话与反馈沉淀，供个人复盘与运营优化。

### 4.2 业务主链路（考试）

1. 企业维护题库与评分维度口径。
2. 配置并发布考试类智能体（绑定题库、题量与达标规则等）。
3. 用户进入考试，在对话形态下完成题目集合。
4. 系统生成成绩与历史记录，管理侧可查询与导出。
5. 与培训运营动作衔接（催促、复考等可在制度或活动层定义，不在本文展开）。

### 4.3 对象之间的关系（对照）

- **题库**同时服务陪练与考试；**智能体类型**决定本次会话的业务目标是巩固还是评定。
- **成绩查询**主要附着在**考试类**智能体；陪练强调过程，是否落库为「成绩」以配置与制度为准。
- **会话**是二者共同载体；**反馈**为质量改进信号，不自动改变题库权威答案。

## 5. 业务规则

### 5.1 显性规则

- 依赖题库的调研/考试/陪练类配置在业务上要求**题库已选定且可用**；否则无法构成有效发布。
- 分值系数类参数在业务上不允许为「零意义」配置（与零分口径混淆），具体以系统校验为准。
- 考试类成绩与导出通常存在条数上限类约束，用于控制批量导出风险。
- 复制产生的智能体副本默认处于需复核状态，避免未校验配置直接对外使用。

### 5.2 隐性规则

- **陪练**：同一题库可多次进入；随机性或顺序性由配置决定；强调「练会」而非「一次定终身」。
- **考试**：强调题集完成度与达标线；更适合作为阶段性检查或活动节点。
- **交互形态相同 ≠ 业务相同**：调研、考试、陪练在移动端可能共用对话形态，但业务目标与统计口径必须区分。

### 5.3 权限或范围限制

- 无权限用户无法参加受限调研或考试类智能体；提示属于权限边界而非系统故障。
- 成绩查询、导出等能力通常仅对智能体管理员或企业管理员开放，不等同全员可见。

### 5.4 特殊说明或例外

- 语音、图片等多模态输入提升贴近真实表达，但判分与解析仍以题库与规则配置为准。
- 与 TBS 训战并存时，建议制度层约定：何种能力用 CMS 快测、何种能力用 TBS 情境演练，避免重复考核口径。

## 6. 常见歧义与边界

### 6.1 容易混淆的概念

- **陪练 vs 考试**：前者重巩固与过程；后者重结果与达标。
- **练习得分 vs 考试成绩**：前者偏过程反馈；后者偏阶段性评定；不可混读为同一人事口径。
- **题库 vs 智能体**：前者是内容资产；后者是交付与规则封装。

### 6.2 容易误解的规则

- **对话很流畅 ≠ 业务目标达成**：考试是否通过仍以成绩与规则为准。
- **移动端与桌面端体验一致 ≠ 权限一致**：权限以账号与可见范围为准。

### 6.3 理解边界

- 本文不写具体题型清单、不设默认通关分；一律以企业当期配置与制度为准。
- 题目「能力维度」标签体系若与绩效或晋升挂钩，应在人力资源或培训制度中单独立法，本文仅描述产品能力边界。

| --- | --- |
| 陪练 | 基于题库、以对话形态为主的巩固式训练智能体类型。 |
| 考试 | 基于题库、以对话形态为主并沉淀成绩的考核类智能体类型。 |
| 题库 | 题目与答案及元数据的集合，为陪练与考试提供内容来源。 |
| 题型 | 区分题目形态与评分权重的配置维度。 |

| 会话 | 用户与智能体交互的过程记录。 |
