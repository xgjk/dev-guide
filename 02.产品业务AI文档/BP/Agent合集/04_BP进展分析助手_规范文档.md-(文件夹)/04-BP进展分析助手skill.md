## Skill 1：阶段输入数据获取
### 1. 基本信息
- Skill 名称：阶段输入数据获取
- Skill 编码：BP04-PROG-001
- 版本号：V1.0
- 所属业务域：BP系统-执行进度分析
- 负责人：BP 进展分析负责人
- 适用 Agent 列表：BP 进展分析助手（内部使用）

### 2. Skill 定位
- 业务目的：为一次“阶段进展分析”准备源数据输入视图
- 解决什么动作：查询并整理源数据（BP 内容 + 关联汇报 + 目标/成果/关键举措关联关系 + 历史阶段记录）并裁剪到阶段窗口
- Skill 类型：查询类 Skill

### 3. 输入输出契约（自然语言）
- 输入参数清单
  - BP 标识：必填
  - 阶段/期次标识 stageId：必填
  - 时间范围 timeRange：选填
  - 用户上下文 userContext：必填（至少含 userId、角色 role、部门 department）
  - 过滤条件 filters：选填（onlyRisks、confidenceMin）
- 参数类型、必填/选填
  - BP 标识：文本型，必填
  - stageId：文本型，必填
  - timeRange：日期区间型，选填
  - userContext：对象型，必填
  - filters：对象型，选填
- 参数校验规则
  - stageId 缺失：直接失败（数据/上下文无法定位）
  - timeRange 若提供：取与阶段窗口的交集作为有效分析窗口；无交集则口径冲突
  - userContext 关键字段非空：用于权限校验与口径判断
- 输出结构
  - 阶段上下文：阶段标识、阶段开始日期、阶段结束日期（用于后续滞后与无动作窗口口径）
  - 关联汇报列表：阶段窗口内的汇报集合（至少能提供可追溯汇报标识）
  - 目标/成果/关键举措实体集合：用于逐实体判断的实体列表
  - 目标-成果-关键举措关联关系整理结果：用于把汇报证据归到正确实体
  - 历史阶段记录：用于“已完成/进行中/未开始”的依据
  - 关联线索（association hints）：把“关联关系”和“证据/汇报”对齐的线索集合
- 错误码结构
  - E_BP04_PARAM_INVALID：参数错误
  - E_BP04_DATA_MISSING：数据缺失（阶段上下文/必要源数据缺失）
  - E_BP04_API_TIMEOUT：上游获取超时
  - E_BP04_PERMISSION_DENIED：权限不足
  - E_BP04_RULE_CONFLICT：口径冲突（如 timeRange 与阶段窗口无交集）
  - E_BP04_INTERNAL_ERROR：内部不可预期错误
- 失败返回格式（自然语言描述）
  - 返回请求标识、错误码、错误信息、错误细节（失败能力/字段）、以及追踪标识 traceId
  - 你选定的策略：获取类能力失败/超时直接失败，不做部分成功

### 4. 前置条件
- 调用前业务条件
  - stageId 必须能定位到对应 bpId 与阶段窗口
  - 调用者对该 BP/阶段数据具备读取权限
- 依赖上下文
  - userContext
  - bpId、stageId
- 依赖权限
  - 只读读取权限
- 依赖外部数据
  - BP 主数据、阶段主数据（起止日期）
  - 关联汇报数据
  - 目标/成果/关键举措与其关联关系数据
  - 历史阶段记录

### 5. 执行逻辑说明
- 主要处理流程
  - 参数校验与权限校验
  - 获取阶段起止日期与最终分析窗口（阶段窗口与 timeRange 交集）
  - 拉取并过滤阶段窗口内的关联汇报
  - 获取目标/成果/关键举措实体与目标-成果-举措关联关系
  - 拉取历史阶段记录
  - 形成后续 Skill 可用的“阶段上下文 + 实体 + 关联线索 + 历史依据”
- 是否调用多个 API：是（用于阶段边界、汇报、实体、关联关系、历史记录）
- 是否有数据清洗、聚合、转换：是（窗口过滤、字段归一、证据标识准备）
- 是否有规则引擎判断：否
- 是否涉及缓存：否

### 6. 所依赖的业务 API 清单
> 先按业务能力占位描述，后续你们替换为真实接口名即可。
- BP获取能力：用于定位 BP 数据域与校验
- 阶段起止获取能力：用于获取阶段开始/结束日期
- 关联汇报获取能力：用于拉取阶段窗口内汇报并保留可追溯汇报标识
- 目标/成果/关键举措关联关系获取能力：用于构建 entityMap 与关联线索
- 历史阶段记录获取能力：用于后续 completed/not_started 的依据

### 7. 结果可信度与解释
- 输出是否需要附证据：需要（汇报标识/历史记录标识用于追溯）
- 是否返回来源字段：需要（为后续 sources/evidence 组装做准备）
- 是否返回口径说明：需要（阶段窗口裁剪口径）
- 是否支持追溯原始数据：需要（所有解释最终回到本 Skill 准备的汇报/历史标识）

### 8. 安全与权限
- 只读
- 是否涉及脱敏：不在本 Skill 内处理（以数据源保证为前提）
- 是否记录审计日志：需要
- 是否要求二次确认：不需要
- 是否要求审批令牌：不需要

### 9. 异常处理
- 参数错误：返回 E_BP04_PARAM_INVALID
- 数据缺失：返回 E_BP04_DATA_MISSING
- API 超时：返回 E_BP04_API_TIMEOUT
- 权限不足：返回 E_BP04_PERMISSION_DENIED
- 业务规则冲突：返回 E_BP04_RULE_CONFLICT
- 部分成功如何返回：不适用（获取类 API 失败直接失败）

### 10. 测试与验收
- 单元测试样例
  - stageId 缺失：直接失败
  - timeRange 与阶段窗口无交集：返回口径冲突
  - 汇报为空：仍成功返回阶段上下文与实体集合，但后续可信度可能下降
- 业务场景测试样例
  - 正常阶段：sources 可追溯标识齐全
- 边界/异常
  - 阶段起止日期缺失：整体数据缺失
- 回归测试要求
  - 确保窗口裁剪逻辑不漂移
  - 确保证据标识字段始终可用

## Skill 2：目标成果举措与汇报关联构建
### 1. 基本信息
- Skill 名称：目标成果举措与汇报关联构建
- Skill 编码：BP04-PROG-002
- 版本号：V1.0
- 所属业务域：BP系统-执行进度分析
- 负责人：BP 进展分析负责人
- 适用 Agent 列表：BP 进展分析助手（内部使用）

### 2. Skill 定位
- 业务目的：把源数据中的“目标/成果/关键举措关联关系”与“汇报证据”对齐到实体粒度
- 解决什么动作：证据归属、字段归一、证据引用组织
- Skill 类型：计算类 Skill

### 3. 输入输出契约（自然语言）
- 输入参数清单
  - stageInput：必填（来自 Skill 1：阶段上下文、汇报列表、实体集合、关联线索、历史记录）
  - filters：选填（onlyRisks、confidenceMin）
- 参数校验规则
  - stageInput 必须包含实体集合与证据归属所需的关联线索
  - 若实体映射无法建立最小证据归属：整体返回数据缺失
  - 若单个实体无法归属：允许该实体进入“未知/证据不足”，不整体失败
- 输出结构
  - 关联映射 associationMap：以每个目标/成果/关键举措实体为粒度，输出其可追溯证据集合（汇报标识/历史记录标识等）
  - 归一后的实体集合 normalizedEntities：对实体相关字段进行口径清洗，确保后续判断可用
- 错误码结构：同统一集合
- 失败返回格式（自然语言描述）
  - 输入结构破损或关键归属线索缺失：整体失败并返回错误码与 traceId
  - 证据不足通常不整体失败，而是实体级 unknown

### 4. 前置条件
- 必须已完成 Skill 1 输出
- stageInput 内关联关系与证据标识可用

### 5. 执行逻辑说明
- 主要处理流程
  - 使用关联线索把汇报证据归属到目标/成果/关键举措实体
  - 对实体字段做口径归一与清洗
  - 组织证据引用集合，确保后续能追溯
- 是否调用多个 API：否
- 是否有数据清洗、聚合、转换：是（归属映射与字段归一）
- 是否有规则引擎判断：否
- 是否涉及缓存：否

### 6. 所依赖的业务 API 清单
- 无（仅处理输入数据）

### 7. 结果可信度与解释
- 输出是否需要附证据：需要（证据归属结果是后续解释的基础）
- 是否返回来源字段：由证据引用承担（后续 Skill 6 组装 sources）
- 是否返回口径说明：需要（是否是“硬归属/软归属”口径说明，避免误读）
- 是否支持追溯原始数据：支持（证据引用可追溯到汇报/历史标识）

### 8. 安全与权限
- 只读
- 记录审计日志
- 不需要二次确认/审批令牌

### 9. 异常处理
- stageInput 关键字段缺失：整体失败（E_BP04_DATA_MISSING）
- 单实体无法归属：实体级 unknown，整体成功

### 10. 测试与验收
- 单元测试样例
  - 关联线索缺失：整体失败
  - 某实体证据不足：仅该实体 unknown
- 业务场景测试样例
  - 同一汇报可影响多个实体：应按你们系统关联粒度归属
- 回归测试要求
  - 证据归属稳定性（同样输入应得同样归属结果）

## Skill 3：阶段进展状态与滞后识别
### 1. 基本信息
- Skill 名称：阶段进展状态与滞后识别
- Skill 编码：BP04-PROG-003
- 版本号：V1.0
- 所属业务域：BP系统-执行进度分析
- 负责人：BP 进展分析负责人
- 适用 Agent 列表：BP 进展分析助手（内部使用）

### 2. Skill 定位
- 业务目的：对每个目标/成果/关键举措在阶段窗口内给出进展状态，并识别滞后与无动作
- 解决什么动作：进展状态分类、滞后分级、无动作窗口判定
- Skill 类型：分析类 Skill

### 3. 输入输出契约（自然语言）
- 输入参数清单
  - associationInput：必填（来自 Skill 2：逐实体证据归属与实体集合）
  - stageContext：必填（来自 Skill 1：阶段开始/结束口径）
  - filters：选填（onlyRisks/confidenceMin）
- 参数校验规则
  - stageContext 必须能确定阶段开始与结束日期
  - 对证据不足的实体：输出 unknown，不整体失败
- 输出结构
  - progressByEntity：逐实体进展判断结果
    - 进度状态：未开始/进行中/按期/轻度滞后/严重滞后/已完成/未知
    - 滞后信息：若为滞后则包含滞后严重度与原因（原因必须写明口径）
    - 无动作信息：无动作标记与原因（必须写明 30 天窗口与证据覆盖口径）
    - 未知原因：缺少哪些关键字段/证据类型
- 错误码结构：同统一集合
- 失败返回格式（自然语言描述）
  - 缺 stageContext：整体失败并返回数据缺失
  - 单实体 unknown：整体仍成功

### 4. 前置条件
- 已完成 Skill 2 的证据归属
- stageContext 可用

### 5. 执行逻辑说明
- 主要处理流程（固定口径）
  - 已完成判定：采用系统“完成标记/完成判定字段”
  - 未开始判定：以阶段开始后 7 天为窗口基准；窗口内如无任何可追溯推进/实施/完成证据，则判未开始
  - 滞后判定：以日期偏差为主
    - 轻度滞后：偏差天数不超过 14 天
    - 严重滞后：偏差天数超过 14 天
  - 无动作判定：以“阶段结束前后最近 30 天”为窗口
    - 无动作口径：硬缺失或软缺失任一满足即算无动作
- 是否调用多个 API：否
- 是否有数据清洗、聚合、转换：是（证据覆盖窗口裁剪/统计）
- 是否有规则引擎判断：否（固定阈值规则）
- 是否涉及缓存：否

### 6. 所依赖的业务 API 清单
- 无

### 7. 结果可信度与解释
- 是否附证据：实体级原因必须能落回证据引用（后续 Skill 6 生成 sources）
- 是否返回来源字段：由 Skill 6 统一承载；本 Skill 输出“原因口径 + 证据缺失类型”
- 是否返回口径说明：必须（7 天未开始窗口、14 天阈值、30 天无动作窗口）
- 是否支持追溯原始数据：是（unknown/结论需说明缺失点，便于定位证据）

### 8. 安全与权限
- 只读
- 记录审计日志
- 不需要二次确认/审批令牌

### 9. 异常处理
- 缺 stageContext：E_BP04_DATA_MISSING
- 缺关键日期字段：实体 unknown
- 部分成功：整体成功，实体级 unknown 不影响其它实体

### 10. 测试与验收
- 单元测试样例
  - 日期偏差 10 天：轻度滞后
  - 日期偏差 20 天：严重滞后
  - 阶段开始后 7 天无证据：未开始
  - 阶段结束最近 30 天证据不可追溯：无动作
- 边界/异常
  - 阶段结束日期缺失：整体失败
- 回归测试要求
  - 阈值与窗口不漂移

## Skill 4：阶段风险点提炼（已删除结构化输入字段的版本）
### 1. 基本信息
- Skill 名称：阶段风险点提炼
- Skill 编码：BP04-PROG-004
- 版本号：V1.0
- 所属业务域：BP系统-执行进度分析
- 负责人：BP 进展分析负责人
- 适用 Agent 列表：BP 进展分析助手（内部使用）

### 2. Skill 定位
- 业务目的：把风险线索提炼成可管理的风险点清单，并绑定影响范围与可追溯证据
- 解决什么动作：风险线索收集、风险归类、严重度标注、风险-实体绑定、风险去重归并
- Skill 类型：分析类 Skill

### 3. 输入输出契约（自然语言）
- 输入参数清单
  - riskInput：必填（来自源数据与前置判断链路的风险线索集合）
  - filters：选填（onlyRisks/confidenceMin）
- 参数类型、必填/选填
  - riskInput：对象型，必填
  - filters：对象型，选填
- 参数校验规则
  - riskInput 中必须提供“可判断风险的依据”（字段清单先不展开，属于待补充项）
  - 已删除/不使用的结构化风险判断输入字段：不再写入输入契约，也不参与判断（ncy、schedule、resource、quality 相关先删除）
  - 若风险线索依据不足：允许返回空风险点或降低可信度（不整体失败）
  - 若单条风险缺证据或严重度：该风险点标注 unknown/证据不足并降低可信度
- 输出结构
  - riskPoints：风险点列表
    - 风险类别：先按你们最终枚举体系输出（本稿不展开，待你补齐；但输出字段必须可用）
    - 风险严重度：low/mid/high 或 unknown
    - 风险一句话总结：面向管理表达
    - 影响范围：关联哪些目标/成果/关键举措实体
    - 证据引用：必须能追溯到汇报/历史记录标识（用于 Skill 6 sources）
  - 风险去重归并结果：相似风险按证据域归并，减少重复
- 错误码结构
  - E_BP04_DATA_MISSING：仅在 riskInput 结构破损到无法读取任何“风险线索依据”时使用
  - 其它同统一集合
- 失败返回格式（自然语言描述）
  - 输入结构破损导致无法识别任何风险线索：整体失败返回错误码与 traceId
  - 风险为空/部分风险 unknown：整体成功，可信度降低

### 4. 前置条件
- riskInput 已包含可用的风险线索依据（待补充字段清单）
- 与实体关联线索存在（用于风险影响范围绑定）
- 读取权限足够（只读）

### 5. 执行逻辑说明
- 主要处理流程（不依赖你删除的 4 类结构化字段）
  - 从 riskInput 提取风险线索依据
  - 将风险线索归类为风险类别（类别体系待补齐口径，但输出字段必须一致）
  - 标注严重度（依据 riskInput 中可识别的严重度信息；缺失则 unknown）
  - 绑定风险影响实体
  - 绑定可追溯证据引用并完成风险去重归并
- 是否调用多个 API：否
- 是否有数据清洗、聚合、转换：是（归并去重、证据域整理）
- 是否有规则引擎判断：否（固定映射/字段可用性判断为主）
- 是否涉及缓存：否

### 6. 所依赖的业务 API 清单
- 无

### 7. 结果可信度与解释
- 是否附证据：需要（风险点必须带可追溯证据引用）
- 是否返回来源字段：证据引用在输出中给出；sources 在 Skill 6 统一组装
- 是否返回口径说明：需要（说明严重度/类别来源是“风险线索依据中的字段”还是“缺失导致 unknown”）
- 是否支持追溯原始数据：支持

### 8. 安全与权限
- 只读
- 记录审计日志
- 不需要二次确认/审批令牌
- 不做本 Skill 脱敏

### 9. 异常处理
- riskInput 无任何风险线索依据：E_BP04_DATA_MISSING（整体失败）
- 单风险点缺证据：该风险点 unknown/证据不足，整体成功
- 部分成功如何返回：整体成功，风险清单可为空或包含 unknown

### 10. 测试与验收
- 单元测试样例
  - riskInput 仅有风险描述但无严重度：输出严重度=unknown
  - 同一证据域多条相似风险：应归并减少数量
- 业务场景测试样例
  - 能从滞后/无动作证据中抽取风险点并绑定实体与证据
- 边界/异常
  - riskInput 为空或破损：整体失败或返回空风险点（按你们最终选择；本稿按“破损到无法读取依据才失败”）
- 回归测试要求

## Skill 6：可信度与证据解释编排（最终对外输出）
### 1. 基本信息
- Skill 名称：可信度与证据解释编排
- Skill 编码：BP04-PROG-006
- 版本号：V1.0
- 所属业务域：BP系统-执行进度分析
- 负责人：BP 进展分析负责人
- 适用 Agent 列表：BP 进展分析助手（内部使用）

### 2. Skill 定位
- 业务目的：把判断结果与证据来源、口径说明、可信度合并成最终可追溯输出
- 解决什么动作：证据挂载、可信度计算、解释字段生成、最终输出整装
- Skill 类型：编排类 Skill

### 3. 输入输出契约（自然语言）
- 输入参数清单
  - finalInput：必填（阶段上下文、逐实体进度判断、风险点、管理输出）
  - filters：选填（confidenceMin）
- 参数校验规则
  - 必须能生成最终输出核心内容（至少 summary 与 confidence）
  - 必须能从链路中提取证据引用；若完全无法生成 sources：整体失败
- 输出结构
  - 最终结果展示内容：summary、laggingItems、noActionItems、riskPoints、focusQuestions、confidence

    - 阶段窗口口径
    - 滞后阈值口径（轻度<=14天、严重>14天）

- 错误码结构
  - E_BP04_DATA_MISSING：当 sources 无法生成或关键输入破损导致无法追溯时使用
  - 其它同统一集合
- 失败返回格式（自然语言描述）
  - 返回请求标识 + 错误码 + 错误细节 + traceId
  - 你选择的证据刚性要求：最终无法生成 sources 则整体失败

- 已拥有证据引用线索（来自 Skill 1~4 链路）
- finalInput 可用于输出整装

### 5. 执行逻辑说明

  - 生成口径说明与追溯解释
  - 按 confidenceMin 处理低可信条目（标注或排除，按你们最终展示策略）
- 是否调用多个 API：否
- 是否有数据清洗、聚合、转换：是（sources 去重、条目筛选/标注）
- 是否有规则引擎判断：否
- 是否涉及缓存：否

### 6. 所依赖的业务 API 清单
- 无

### 7. 结果可信度与解释
- 是否附证据：是（刚性要求）
- 是否返回来源字段：是（sources 必须返回）
- 是否返回口径说明：是（caliber 必须返回）
- 是否支持追溯原始数据：是（traceability + sources）

### 8. 安全与权限
