## 一、概述

基础服务包含了全系统的底层支撑能力，涵盖身份、资产、标签及 AI 辅助实体管理：

1. **用户服务**：组织架构、人员信息及搜索服务。
2. **文件服务**：物理文件存储、分片上传及下载加速。
3. **AI机器人管理**：三方机器人通道的注册与可见性维护。
4. **标签管理**：面向工作的个人分类标记体系。
5. **虚拟员工管理**：真实用户关联的数字助手（NPC）实体管理。

本文档作为入口索引，引导 AI 和开发者快速定位具体的接口区域。

## 四、核心业务流程

### 4.1 身份与组织同步
涉及搜索员工后，深度拉取其所在的二三级部门及汇报线。
> 详情见：[01-用户服务.md](API接口明细/01-用户服务.md)

### 4.2 资产存储与流转
涉及利用物理通道或云 Token 上传文件，支持大文件分片上传与秒传（MD5 预检），并获取加密下载外链。
> 详情见：[02-文件服务.md](API接口明细/02-文件服务.md)

### 4.3 AI 助理接入
涉及到注册机器人通道后，将其与特定用户或虚拟员工身份进行绑定。
> 详情见：[03-AI机器人管理.md](API接口明细/03-AI机器人管理.md)

### 4.4 个人化助理与标签管理
用户可以创建虚拟员工身份（NPC），并为其打上个人标签进行分类，最后通过 AI 机器人接口完成能力的生产力转化。
> 详情见：[04-标签管理.md](API接口明细/04-标签管理.md) 和 [05-虚拟员工管理.md](API接口明细/05-虚拟员工管理.md)

## 五、错误码说明

开放平台接口统一使用 `resultCode` 表示业务处理结果：

| resultCode | 说明 | 推荐处理动作 |
| ---------- | ---------------------- | ------------------------------------------ |
| **1** | **请求成功** | 继续后续业务逻辑 |
| **0** | **通用失败** | 读取 `resultMsg` 提示用户 |
| **500** | **系统异常** | 稍后重试 |
| 610012 | **请求太频繁（限流）**| 触发指数退避重试 |
| 610015 | 无访问权限 | 联系管理员检查 AppKey 授信范围 |

## 六、注意事项

1. **ID 精度保护**：对于 `Long` 类型 ID，前端及三方对接务必使用 **String** 接收，严禁使用 Number 承载以防截断。
2. **下载链接失效性**：文件外链通常存在 **1 小时** 的 TTL，不建议在业务数据库中持久化存储外链字符串。
3. **API 规范合规**：所有接口均包含 `AI决策` 列，AI 提示词工程（Prompt Engineering）应优先参考该列决定字段选用。
