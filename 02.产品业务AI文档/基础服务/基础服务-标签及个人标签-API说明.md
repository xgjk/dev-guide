## 一、概述

本文档描述了 **标签服务** 对外开放的 Open API 接口。个人标签允许用户在工作协同系统中对汇报进行分类管理，通过这些接口，可以实现以下业务能力：

1. **个人标签管理**：
   - **我的个人标签列表** — 获取当前账号下已定义的全部个人标签。
   - **创建个人标签** — 新增一个自定义标签名称。
   - **更新个人标签** — 修改已有标签的名称。
   - **删除个人标签** — 物理删除指定的标签定义（仅删除标签本身）。

## 二、通用说明

### 2.1 访问地址
```
https://{域名}/open-api/{接口地址}
```

### 2.2 环境信息

| 环境 | 域名/Base URL | 备注 |
| ---- | ------------------------------ | --- |
| 生产环境 | `https://sg-al-cwork-web.mediportal.com.cn` | - |

### 2.3 公共请求头

| Header | 说明 | 是否必填 |
| -------- | -------------------------- | ----- |
| `appKey` | 从工作协同系统获取的 API 密钥 | 是 |

### 2.4 通用响应结构

所有接口响应均返回统一的 `Result<T>` 结构：

```json
{
  "resultCode": 1,
  "resultMsg": null,
  "data": null
}
```

| 字段 | 类型 | 说明 |
| ------------ | ------- | ----------------------------------------- |
| `resultCode` | Integer | 业务状态码，`1` 表示成功，非 `1` 表示失败 |
| `resultMsg`  | String  | 提示信息，成功时为 `null`，失败时为错误描述 |
| `data`       | T       | 业务数据，根据各接口不同，失败时通常为 `null` |

## 三、关键业务流程说明

### 场景一：管理并使用个人标签

> 需求：用户希望通过 API 定义一套自己的标签体系，以便后续在发汇报或管理收件箱时进行快速分类。

1. 调用 **4.1 我的个人标签列表**（`GET /work-report/label/listLabel`），查看当前已有的标签及其 `id`。
2. 若需新标签，调用 **4.2 创建个人标签**（`POST /work-report/label/createLabel`），传入 `name="项目A"`，获取返回的标签对象（含 `id`）。
3. 如需更名，调用 **4.3 更新个人标签**（`POST /work-report/label/updateLabel`），传入 `id` 和新的 `name`。
4. 若标签不再需要且尚未深度绑定，调用 **4.4 删除个人标签**（`GET /work-report/label/delLabel`）进行清理。

> **注意**：本接口仅处理标签定义的维护。关于如何将标签与具体汇报“绑定”或“解绑”，请参考 **《工作协同 API 说明》** 中的相关章节。

## 四、标签服务接口详细说明

### 4.1 我的个人标签列表

获取当前登录用户创建的所有个人标签列表。

**基本信息**

| 项目 | 说明 |
| ------------ | ----------------------------------- |
| 接口地址 | `/work-report/label/listLabel` |
| 请求方式 | `GET` |

**请求参数**

无。

**响应参数**

`data` 类型为 `List<ReportLabelVO>`，详见 **[5.1 ReportLabelVO](#51-reportlabelvo)**。

**请求示例**

```bash
curl -X GET 'https://{域名}/open-api/work-report/label/listLabel' \
  -H 'appKey: YOUR_API_KEY'
```

**响应示例**

```json
{
  "resultCode": 1,
  "resultMsg": "成功",
  "data": [
    {
      "id": 1234567890123,
      "name": "待处理",
      "createTime": "2026-04-01 10:00:00"
    }
  ]
}
```

---

### 4.2 创建个人标签

新增一个自定义个人标签。

**基本信息**

| 项目 | 说明 |
| ------------ | ----------------------------------- |
| 接口地址 | `/work-report/label/createLabel` |
| 请求方式 | `POST` |
| Content-Type | `application/json` |

**请求参数**

`ReportLabelOperationParam`：

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `name` | String | 是 | 标签名称 |

**响应参数**

`data` 类型为 `ReportLabelVO`，返回新增成功的标签详细信息。

**请求示例**

```bash
curl -X POST 'https://{域名}/open-api/work-report/label/createLabel' \
  -H 'appKey: YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "重要事项"
  }'
```

---

### 4.3 更新个人标签

修改已有个人标签的名称。

**基本信息**

| 项目 | 说明 |
| ------------ | ----------------------------------- |
| 接口地址 | `/work-report/label/updateLabel` |
| 请求方式 | `POST` |
| Content-Type | `application/json` |

**请求参数**

`ReportLabelOperationParam`：

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `id` | Long | 是 | 标签 ID |
| `name` | String | 是 | 新的标签名称 |

**响应参数**

`data` 类型为 `Boolean`，成功返回 `true`。

---

### 4.4 删除个人标签 (仅删除标签)

物理删除指定的个人标签定义。注意：该操作仅删除标签本身，不会影响已关联的汇报内容，但关联关系会失效。

**基本信息**

| 项目 | 说明 |
| ------------ | ----------------------------------- |
| 接口地址 | `/work-report/label/delLabel` |
| 请求方式 | `GET` |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `id` | Long | 是 | 标签 ID |

**响应参数**

`data` 类型为 `Boolean`，成功返回 `true`。

---

## 五、公共数据结构

### 5.1 ReportLabelVO

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | Long | 标签/绑定关系 ID |
| `name` | String | 标签名称 |
| `labelId` | Long | 标签正式 ID（为空时代表当前对象即为标签定义本身） |
| `reportRecordId` | Long | 关联的汇报 ID（仅在查询绑定关系时有值） |
| `createTime` | Timestamp | 创建时间 |
