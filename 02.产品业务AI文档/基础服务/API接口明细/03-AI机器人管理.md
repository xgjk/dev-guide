## 二、典型业务流程

### 场景：创建并发布一个私人助理机器人

1. 调用 **3.1 注册 AI 机器人**（`POST /im/robot/register`），提交基础信息。
2. 接口返回 `RobotPluginVO`，其中包含对接所需的认证信息与连接地址。
3. 调用 **3.2 获取我的私人助理**（`GET /im/robot/getMyRobot`）校验状态。

## 三、接口详细说明

### 3.1 注册 AI 机器人

注册一个新的 AI 机器人实体，并获取其专属的连接配置。

**基本信息**

| 项目 | 说明 |
| ------------ | ----------------------------------- |
| 接口地址 | `/im/robot/register` |
| 请求方式 | `POST` |
| Content-Type | `application/json` |
| 数据量级别 | 小（单条记录） |

**请求参数**

- **请求体 (Body)：`RobotRegisterRequest`**

| 参数名 | 类型 | 必填 | 说明 |
| ----------- | ------------- | ---- | ------------------------------ |
| `agentId` | String | 是 | 绑定的外部(openclaw)AgentID |
| `name` | String | 是 | 机器人名称 |
| `avatar` | String | 否 | 机器人头像 URL |
| `groupLabel` | String | 否 | 分组标签 |
| `remark` | String | 否 | 备注信息 |
| `visibleType` | Integer | 否 | 0-私有, 1-公开, 2-指定范围 |
| `visibleRange` | List\<VisibleRangeItem\> | 否 | `visibleType=2` 时必传 |

**响应参数**

`data` 类型为 `RobotPluginVO`，详见 **[4.2](#42-robotpluginvo)**。

**请求示例**

```bash
curl -X POST 'https://{域名}/open-api/im/robot/register' \
  -H 'appKey: YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "agentId": "main",
    "name": "销售助手",
    "visibleType": 0
  }'
```

**响应示例**

```json
{
  "resultCode": 1,
  "resultMsg": "success",
  "data": {
    "agentId": "main",
    "appKey": "robot_app_key_xxx",
    "userId": "734612345678901234",
    "baseUrl": "https://api.example.com",
    "wsBaseUrl": "wss://api.example.com"
  }
}
```

### 3.2 获取我的私人助理

获取当前登录账号权限范围内可见的机器人列表。

**基本信息**

| 项目 | 说明 |
| ------------ | ----------------------------------- |
| 接口地址 | `/im/robot/getMyRobot` |
| 请求方式 | `GET` |
| 数据量级别 | 小（通常 ≤ 10 条） |

**请求参数**

无。

**响应参数**

`data` 类型为 `List<AiRobotVO>`，详见 **[4.1](#41-airobotvo)**。

**请求示例**

```bash
curl -X GET 'https://{域名}/open-api/im/robot/getMyRobot' \
  -H 'appKey: YOUR_API_KEY'
```

---

### 3.3 根据 AgentId 删除我的机器人

根据机器人唯一标识 `agentId` 下线并物理删除机器人配置。

**基本信息**

| 项目 | 说明 |
| ------------ | ----------------------------------- |
| 接口地址 | `/im/robot/deleteMyRobot` |
| 请求方式 | `POST` |
| 数据量级别 | 小 |

**请求参数**

| 参数名 | 位置 | 类型 | 必填 | 说明 |
| ----------- | ------ | ---- | ---- | ------------------------------ |
| `agentId` | Query | String | 是 | 机器人唯一标识 |

**响应参数**

`data` 类型为 `Boolean`，表示是否操作成功。

## 四、公共数据结构

### 4.1 AiRobotVO

| 字段 | 类型 | AI决策 | 说明 |
| ------------- | ------ | ----- | ------------------- |
| `agentId` | String | 是 | Agent ID |
| `appKey` | String | 是 | 机器人的认证 AppKey |
| `userId` | String | 是 | 机器人虚拟用户 ID |
| `name` | String | 是 | 机器人名称 |
| `avatar` | String | 否 | 头像 URL |
| `groupLabel` | String | 否 | 分组标签 |
| `remark` | String | 否 | 备注 |

### 4.2 RobotPluginVO

| 字段 | 类型 | AI决策 | 说明 |
| ------------- | ------ | ----- | ------------------- |
| `agentId` | String | 是 | Agent ID |
| `appKey` | String | 是 | 机器人的认证 AppKey |
| `userId` | String | 是 | 虚拟用户 ID |
| `baseUrl` | String | 是 | 后台服务 HTTP 域名 |
| `wsBaseUrl` | String | 是 | 后台服务 WSS 域名 |
| `name` | String | 否 | 机器人名称 |

### 4.3 VisibleRangeItem

| 字段 | 类型 | AI决策 | 说明 |
| ------------- | ------ | ----- | ------------------- |
| `targetId` | String | 是 | 部门或人员 ID |
| `targetType` | String | 是 | `USER`-人员，`DEPT`-部门 |
