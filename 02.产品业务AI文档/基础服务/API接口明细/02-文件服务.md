## 一、能力概述

| 接口 | 说明 | 数据量 |
|------|------|------|
| 4.1 上传整个文件 | 简单上传单个完整文件（20MB以下建议） | 大 |
| 4.2 获取文件下载信息 | 根据 resourceId 获取临时的下载 URL | 小 |
| 4.3 获取七牛上传 Token | 获取七牛云直传所需的上传凭证信息 | 小 |
| 4.4 【分片】预检文件MD5 | 上传前预检，支持秒传判定 | 小 |
| 4.5 【分片】注册分片数据 | 将已物理上传到存储（MinIO）的分片信息在服务端注册 | 小 |
| 4.6 【分片】合并生成资源 | 所有分片注册完成后，触发服务端合并生成 resourceId | 小 |

## 二、典型业务流程

### 场景一：上传普通文件并获取下载链接
> 需求：第三方系统期望向文件服务器推送一个本地文件，并快速获取供前端展示或下载的临时 URL。

1. 调用 **4.1 上传整个文件**（`POST /cwork-file/uploadWholeFile`），获取 `resourceId`。
2. 调用 **4.2 获取文件下载信息**（`GET /cwork-file/getDownloadInfo`），传入 `resourceId`，得到 `downloadUrl`。

### 场景二：大文件分片上传流程 [推荐]
> 需求：针对大文件或网络环境不稳定的场景，通过分段上传确保可靠性。

1. **预检与秒传**：调用 **4.4 预检文件MD5**。
   - 若返回 `sliceId`：说明服务器已存在该内容，直接跳到 Step 4。
   - 若返回 `uploadUrl`：说明文件不存在，记录该 URL。
2. **物理上传（客户端执行）**：客户端使用 `PUT` 请求将文件分片二进制流上传至 Step 1 获取的 `uploadUrl`。需要确保响应状态码为 `200`。
3. **分片注册**：调用 **4.5 注册分片数据**，传入分片的 MD5、大小及物理路径（预检返回的 `fullPath`），换取 `sliceId`。
4. **资源合并**：待所有分片均注册完成，调用 **4.6 合并生成资源**，传入所有 `sliceId`，获取最终的接口入参 **`resourceId`**。

**大文件分片上传 Python 完整代码示例：**

> **🚨 核心要求 (MinIO)**：MinIO 对分片上传的分块大小有严格限制，**单个分片大小最小必须为 5MB**（最后一个分片除外），否则在最后合并时或上传分片时会报错。以下代码严格按照 `5 * 1024 * 1024` (5MB) 进行分块处理。

```python
import os
import hashlib
import requests

# 测试时请替换为你真实的接口域名与鉴权信息
BASE_URL = "https://{域名}/open-api/document-database/file"
HEADERS = {
    "appKey": "YOUR_API_KEY",
    # 视具体的项目鉴权需要添加其他 Header
}

def calculate_md5(data: bytes) -> str:
    """计算二进制流的 MD5"""
    m = hashlib.md5()
    m.update(data)
    return m.hexdigest()

def upload_large_file(file_path: str) -> int:
    """
    大文件分片上传完整流程示例
    """
    # 🚨 MinIO 限制：分片大小最小为 5MB (除了最后一块)
    CHUNK_SIZE = 5 * 1024 * 1024 
    
    file_size = os.path.getsize(file_path)
    file_name = os.path.basename(file_path)
    file_suffix = file_name.split('.')[-1] if '.' in file_name else ''
    slice_ids = []

    print(f"开始传输文件: {file_name}, 总大小: {file_size} 字节")

    with open(file_path, "rb") as f:
        chunk_index = 0
        while True:
            chunk_data = f.read(CHUNK_SIZE)
            if not chunk_data:
                break
                
            chunk_size = len(chunk_data)
            chunk_md5 = calculate_md5(chunk_data)
            print(f"==> 正在处理第 {chunk_index + 1} 个分片，大小: {chunk_size} 字节，MD5: {chunk_md5}")

            # -------------------------------------------------------------
            # Step 1: 预检分片 MD5 (支持秒传)
            # -------------------------------------------------------------
            check_url = f"{BASE_URL}/getSliceIdByMd5V2"
            check_params = {
                "md5": chunk_md5,
                "size": chunk_size,
                "suffix": file_suffix
            }
            resp = requests.get(check_url, params=check_params, headers=HEADERS).json()
            if resp.get("resultCode") != 1:
                raise Exception(f"分片预检失败: {resp}")
            
            slice_data = resp.get("data", {})
            slice_id = slice_data.get("sliceId")
            
            if slice_id:
                print(f"  [命中秒传] 服务器已存在该内容分段，跳过物理上传，sliceId: {slice_id}")
            else:
                upload_url = slice_data.get("uploadUrl")
                full_path = slice_data.get("fullPath")
                
                # -------------------------------------------------------------
                # Step 2: 物理上传分段到物理存储网关 (MinIO)
                # -------------------------------------------------------------
                print(f"  [物理上传] 未命中秒传，正在将分段数据上传至 MinIO预签名链接...")
                # 注意：直传 MinIO 请直接 PUT 数据流，无需带上服务端的 appKey Header，以防跨域或签名校验出错
                put_resp = requests.put(upload_url, data=chunk_data)
                put_resp.raise_for_status() 
                print(f"  [物理上传成功]")
                
                # -------------------------------------------------------------
                # Step 3: 在服务端注册分片的物理元信息
                # -------------------------------------------------------------
                register_url = f"{BASE_URL}/uploadFileSliceV2"
                register_payload = {
                    "filePath": full_path,
                    "md5": chunk_md5,
                    "size": chunk_size,
                    "storageType": slice_data.get("storageType", "MINIO")
                }
                # 注意此处 Header 为 JSON
                reg_headers = {**HEADERS, "Content-Type": "application/json"}
                reg_resp = requests.post(register_url, json=register_payload, headers=reg_headers).json()
                if reg_resp.get("resultCode") != 1:
                    raise Exception(f"注册分片失败: {reg_resp}")
                
                slice_id = reg_resp.get("data")
                print(f"  [注册分片成功] 获得该分段数据库标识 sliceId: {slice_id}")
            
            # 存入列表，留待最后合并使用
            slice_ids.append(slice_id)
            chunk_index += 1

    # -------------------------------------------------------------
    # Step 4: 合并所有已上传或秒传的分片资源
    # -------------------------------------------------------------
    print(f"==> 所有分段处理完毕开始触发资源合并, 累计收集到 {len(slice_ids)} 个 sliceId")
    merge_url = f"{BASE_URL}/saveResource"
    merge_payload = {
        "name": file_name,
        "sliceIds": slice_ids,
        "size": file_size,
        "suffix": file_suffix
    }
    merge_headers = {**HEADERS, "Content-Type": "application/json"}
    merge_resp = requests.post(merge_url, json=merge_payload, headers=merge_headers).json()
    if merge_resp.get("resultCode") != 1:
        raise Exception(f"资源合并失败: {merge_resp}")

    resource_id = merge_resp.get("data")
    print(f"==> 🚀 文件上传整体流程完成，最终业务可用 resourceId: {resource_id}")
    return resource_id

if __name__ == "__main__":
    # 使用示例，指向一个超过 5MB 的本地文件进行测试
    test_file_path = "your_large_video_or_file.mp4"
    if os.path.exists(test_file_path):
        final_resource_id = upload_large_file(test_file_path)
    else:
        print(f"测试文件不存在: {test_file_path}")
```

## 三、接口详细说明

### 4.1 上传整个文件

上传本地完整文件，返回文件资源 ID。

**基本信息**

| 项目 | 说明 |
| ------------ | ----------------------------------- |
| 接口地址 | `/cwork-file/uploadWholeFile` |
| 请求方式 | `POST` |
| Content-Type | `multipart/form-data` |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
| ----------- | ------------- | ---- | ------------------------------ |
| `file` | binary (File) | 是 | 要上传的文件 |

**响应参数**

`data` 类型为 `Long`（resourceId）。

**请求示例**

```bash
curl -X POST 'https://{域名}/open-api/cwork-file/uploadWholeFile' \
  -H 'appKey: YOUR_API_KEY' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@/path/to/your/file.png'
```

---

### 4.2 获取文件下载信息

根据资源 ID 获取文件下载信息。下载链接有效期为 1 小时。

**基本信息**

| 项目 | 说明 |
| ------------ | ----------------------------------- |
| 接口地址 | `/cwork-file/getDownloadInfo` |
| 请求方式 | `GET` |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
| ------------ | ---- | ---- | ----------------- |
| `resourceId` | Long | 是 | 文件资源 ID |

**响应参数**

`data` 类型为 `DownloadFileVO`（见 4.1）。

---

### 4.3 获取七牛上传 Token

获取七牛云直传所需的上传凭证信息。

**基本信息**

| 项目 | 说明 |
| ------------ | ----------------------------------- |
| 接口地址 | `/cwork-file/getUploadToken/cwork` |
| 请求方式 | `GET` |

**响应参数**

`data` 类型为 `QiniuUploadTokenVO`（见 4.2）。

---

### 4.4 【分片】预检文件MD5 (支持秒传)

在大文件上传前，根据分片的 MD5 进行预查。

**基本信息**

| 项目 | 说明 |
| ------------ | ----------------------------------- |
| 接口地址 | `/document-database/file/getSliceIdByMd5V2` |
| 请求方式 | `GET` |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `md5` | String | 是 | 文件分片的 MD5 |
| `size` | Long | 否 | 文件总大小（单位：字节） |
| `suffix` | String | 否 | 文件后缀（如 `pdf`） |

**响应参数**

`data` 类型为 `SliceCheckVO`（见 4.3）。

---

### 4.5 【分片】注册分片数据

当物理上传（PUT 至 MinIO）完成后，在服务端登记该分片。

**基本信息**

| 项目 | 说明 |
| ------------ | ----------------------------------- |
| 接口地址 | `/document-database/file/uploadFileSliceV2` |
| 请求方式 | `POST` |
| Content-Type | `application/json` |

**请求参数**

请求体为 `UploadFileSliceParam`（见 4.4）。

**响应参数**

`data` 类型为 `Long`（分片对应的 `sliceId`）。

---

### 4.6 【分片】合并生成资源

在所有分片注册完成后，调用此接口合并分片为完整资源。

**基本信息**

| 项目 | 说明 |
| ------------ | ----------------------------------- |
| 接口地址 | `/document-database/file/saveResource` |
| 请求方式 | `POST` |
| Content-Type | `application/json` |

**请求参数**

请求体为 `SaveResourceParam`（见 4.5）。

**响应参数**

`data` 类型为 `Long`（最终的 **resourceId**）。

---

## 四、公共数据结构

### 4.1 DownloadFileVO
| 字段 | 类型 | AI决策 | 说明 |
| ------------- | ------ | ------ | ------------------- |
| `downloadUrl` | String | 是 | 下载 URL（有效期 1 小时） |
| `fileName` | String | 是 | 文件名 |
| `resourceId` | Long | 是 | 资源 ID |
| `suffix` | String | 否 | 文件后缀 |
| `size` | Long | 否 | 文件大小（字节） |

### 4.2 QiniuUploadTokenVO
| 字段 | 类型 | AI决策 | 说明 |
| --------- | ------ | ------ | ------------------------ |
| `token` | String | 是 | 七牛上传凭证 |
| `key` | String | 是 | 建议的上传 key（通常建议由前端按 UUID 生成） |
| `host` | String | 是 | 七牛上传 Host 地址 |

### 4.3 SliceCheckVO
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `sliceId` | Long | 切片 ID（若直接命中说明已秒传成功） |
| `uploadUrl` | String | MinIO 预签名上传链接（未命中时返回，需使用 PUT 方式物理上传） |
| `fullPath` | String | 服务端目标存储路径（供 `uploadFileSliceV2` 的 `filePath` 使用） |

### 4.4 UploadFileSliceParam
| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `filePath` | String | 是 | 预检接口返回的 `fullPath` |
| `md5` | String | 是 | 分片的 MD5 值 |
| `size` | Long | 是 | 单个分片大小（字节） |
| `storageType` | String | 是 | 存储类型：例如 `MINIO` |

### 4.5 SaveResourceParam
| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `name` | String | 是 | 文件名称（含后缀） |
| `sliceIds` | array[Long] | 是 | 所有分片对应的 `sliceId` 列表 |
| `suffix` | String | 否 | 文件后缀 |
| `size` | Long | 否 | 文件总大小 |
