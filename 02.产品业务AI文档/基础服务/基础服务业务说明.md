## 一、系统概述
**基础服务系统**是支撑主要业务应用运行的底层核心，主要由 **用户服务（User Service）** 与 **文件服务（File Service）** 两大模块构成。

- **用户服务**：作为身份与权限的唯一真实数据源（SSOT），负责组织架构、人员信息、身份认证以及多端渠道（如内部办公、开放平台、钉钉等）的接入与打通。
- **文件服务**：作为数字资产的中央枢纽，管理所有在业务中产生的非结构化数据（文档、图片、音视频等），涵盖从上传、转换、在线协作到最终归档审计的全生命周期闭环。

通过这两大基础模块的联动，为上层应用提供了“正确的人在安全的环境下访问和流转正确的文档资产”的底层屏障保障。

## 二、系统核心对象
基础服务构建于以下几类高度抽象的核心资源之上：

### 2.1 用户服务核心对象
| 对象 | 业务说明 | 产生/关联 | 用途定位 |
| ---- | ---- | ------------ | -------- |
| **公司/部门** | (Company/Dept) 组织内的行政或业务边界节点 | 员工/下级部门 | 决定数据的天然归属及多租户/物理隔离边界 |
| **员工** | (Employee) 具有特定身份标识的自然人 | 岗位角色/系统行为 | 业务发起、执行与流转的基础原子单元 |
| **职务/岗位** | (Position/Duty) 员工在系统内承接的标准化职责 | 角色权限 | 连接人与系统操作范围的桥梁 |
| **角色** | (Role) 特定权限点的集合 | 操作许可 | 系统鉴权引擎（RBAC）判定行为合法与否的基础 |

### 2.2 文件服务核心对象
| 对象 | 业务说明 | 产生/关联 | 用途定位 |
| ---- | ---- | ------------ | -------- |
| **文件** | (File/Document) 实际的数字业务载体 | 版本/转换记录 | 提供预览、下载和流转的业务核心客体对象 |
| **文件夹** | 组织和聚合相关文件资产的容器 | 子文件(夹) | 构建和沉淀企业知识树或业务归档库 |
| **协作关系** | (Cooperation) 人员针对文件的动态权限映射 | 读写修订记录 | 保障在同一频道的安全知识传递与共创 |
| **操作日志** | (Log) 针对文件任何行为的快照快点 | 审计追踪 | 防止数据资产受损、被窃，可完整还原变更链条 |

## 三、核心业务关系
基础服务虽然在代码工程上被划分为不同的微服务，但在实际业务中保持深度耦合的关系。

**1. 账户认证链路关系：**
```
外部信源 (钉钉/ERP等) 或 内部登录网关
 ↓
进入员工认证体系 (EmpLogin / Sso)
 ↓
绑定 组织架构(Org) 身份
 ↓
分发 Token，附带所持 角色/岗位(Role/Duty) 权限
```

**2. 资产流转与安全关系：**
```
操作者 (发起请求)
 ↓
鉴权网关 (验证 User 服务的 Token 及安全上下文 SecureInner)
 ↓
放行操作 -> [文件服务 (上传/下载/预览/协作)]
 ↓
操作完成 -> 异步生成审计日志与派生格式 (Convert/Voicex)
```

**3. 总业务流闭环：**
```
身份赋能 → 资产创造 → 协同流转 → 审计留存
```

## 四、业务规则

### 规则一：组织是权限的最高上下文
一切角色的效力都依附于该员工所在的组织节点。当员工发生部门调整时，系统依赖 `SyncData` 同步机制切断他原有的上层访问数据链路，重塑新的安全上下文。

### 规则二：身份一元化与多端聚合
无论流量入口来自自研系统Web端、AppKey开放平台，还是三方钉钉（Dingtalk），内部始终收口并映射为一个全局唯一的 Employee 主键，保障资产归属的一致性。

### 规则三：资产流转“安全左移”
任何文件（夹）的访问、分享或协作行为，必须前置调用用户服务的接口校验角色权限，而非等获取到物理文件路径后再进行拦截；核心文档原则上不被“物理销毁”，采用软删除或归档策略（保留 OperationLog）。

### 规则四：格式屏蔽与流体协同
所有被上传的文件（尤其是特殊的业务文档），系统会自动调用转换引擎（FileConvert / Speech识别功能）将其转译为不受客户端环境约束的通用流体格式（如网页H5/音转文），保障信息在任何网络端都能实现无障碍流转。

## 五、主要功能
在这两套核心模型支撑下，目前的系统具备以下关键能力阵列：

### 用户服务 (User Service) 能力图谱：
1. **组织人事管理引擎**：快速响应复杂的入、离、调、转生命周期，提供 `Department`、`Employee`、`OrgStructure` 数据的精准增删改查机制。
2. **多模态合规鉴权**：支持基于强身份体系的 SSO 单点登录与应用（AppKey）白名单授信。
3. **企业微信/钉钉融合对接**：与外部通讯工具无缝整合（考勤、免登、回调、通讯录同步）。

### 文件服务 (File Service) 能力图谱：
1. **全域数字资产储管池**：囊括日常办公文档，并能支撑其重命名、多维度检索分类与精细化移动。
2. **极速上云传输隧道**：借助断点续传、高效分片机制 (`UpDownload`)，实现海量业务附件的高速上云。
3. **多维内容加工工厂**：提供音视频语音提取转写（`Speech` / `Voicex`）、文档转PDF/图片的服务。
4. **防泄漏与在线协同系统**：记录全部协作（`Cooperation`）行为与操作留痕。

## 六、系统运行模式
在全系统微服务架构中，基础系统的角色定位犹如操作系统的“内核态”，通过以下方式维系全局健康：
```
【事件发生态】 拦截每一次资产创建与访问的合规性
       ↓
【常驻静默态】 在后台定期同步全公司的核心架构与字典数据
       ↓
【驱动通知态】 在身份变动或核心资料更新时，全网广播状态重构信号
```
该模式使得在多业务线并行的生态系统中，复杂多变的业务应用都能够建立在清晰稳定的人员信息以及安全受控的文档基座之上。
