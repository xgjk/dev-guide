# 系统管理与日志

本模块涵盖首页仪表盘、系统管理（仅管理员）、日志体系和功能设置等内容。

---

## 1. Dashboard 仪表盘（/，首页）

首页是系统的核心数据概览页面，登录后默认展示。

### 1.1 欢迎区域
- 显示当前登录用户的昵称
- 显示系统当前时间（实时更新）
- 右上角"刷新权限"按钮：手动拉取最新用户权限
- 右上角"上传 XHS Cookie"按钮：用于小红书数据采集

### 1.2 监控中视频统计卡片
- 展示当前正在监控的视频总数
- 支持两种视图切换：
  - **品牌视图**：按品牌维度展示监控数量，带进度条
  - **平台视图**：按平台维度展示监控数量，带色块卡片
- 每 5 分钟自动刷新

### 1.3 视频总数统计卡片
- 展示系统中所有视频的总数
- 同样支持品牌/平台两种视图切换

### 1.4 最新提醒动态
- 自动滚动展示最近的提醒信息
- 提醒类型：
  - **KOL 新作品**：KOL 发布了新视频（绿色图标）
  - **视频监控**：监控视频有新动态（橙色图标）
- 每条提醒显示：提醒内容、类型标签、推送时间
- 鼠标悬停可暂停滚动
- 每 5 分钟自动刷新数据

### 1.5 热门 KOL TOP5
- 按粉丝数排序展示前 5 名 KOL
- 显示：排名徽章（金/银/铜）、头像、昵称、粉丝数、获赞数
- 可手动刷新

### 1.6 近 7 天视频发布趋势
- 折线图展示最近 7 天的视频发布量变化
- 悬停可查看具体日期和发布数量
- 可手动刷新

## 2. 用户管理（/system/users）

> 仅超级管理员可访问

### 2.1 功能说明
管理系统用户账号，配置用户角色和推送订阅。

### 2.2 用户搜索
- 按姓名搜索
- 按角色搜索
- 按钉钉 ID 搜索

### 2.3 角色分配
- 为用户分配不同的角色
- 角色决定用户可访问的功能菜单

### 2.4 推送订阅管理
- 管理用户的钉钉消息推送订阅
- 配置用户接收哪些类型的提醒

### 2.5 撤销订阅
- 取消用户的推送订阅
- 取消后用户不再收到钉钉消息提醒

---

## 3. 权限配置（/system/roles）

> 仅超级管理员可访问

### 3.1 功能说明
管理角色信息和菜单权限，决定不同角色能看到哪些功能。

### 3.2 角色信息编辑
- 修改角色名称
- 修改角色描述

### 3.3 菜单权限勾选
- 菜单按分组展示（如：KOL监控、销售数据、日志等）
- 勾选该角色可见的菜单项
- 未勾选的菜单对该角色不可见

### 3.4 保存权限
- 修改后点击保存按钮生效
- 已登录的用户需刷新权限才能看到变化

---

## 4. 品牌/药品管理（/system/medicines）

> 仅超级管理员可访问

### 4.1 功能说明
管理系统中的品牌（药品）信息，这些品牌信息贯穿整个系统的销售数据和分组管理。

### 4.2 品牌信息字段
- **品牌名称**：品牌的显示名称
- **品牌代码**：品牌的唯一标识代码
- **品牌颜色**：品牌在系统中的专属显示颜色
- **品牌描述**：品牌的补充说明

### 4.3 编辑操作
- 可修改品牌的所有信息
- 品牌颜色会影响 GMV 看板、品牌销售等页面的显示

---

## 5. 预警规则管理（/system/alert-rules）

> 仅超级管理员可访问

### 5.1 功能说明
按平台配置视频监控的预警规则，当视频数据满足条件时自动触发钉钉提醒。

### 5.2 支持平台
- 抖音
- 小红书
- 视频号

### 5.3 规则配置项
- **规则类型**：不同类型的监控规则（如互动量阈值、播放量阈值等）
- **阈值设置**：触发预警的数值条件
- **优先级**：规则的执行优先级（高/中/低）
- **启用/禁用**：可随时开关规则

## 6. 日志体系

系统提供 7 种日志，记录各类操作和事件。

### 6.1 监控日志（/logs/followlogs）
- **内容**：KOL 监控的执行记录
- **用途**：查看监控任务的执行情况，排查监控异常

### 6.2 数据中心（/logs/logs）
- **内容**：系统推送的提醒日志
- **用途**：查看所有提醒的历史记录，包括 KOL 新作品提醒和视频监控提醒

### 6.3 同步日志（/logs/sync-logs）
- **内容**：数据同步操作记录
- **筛选**：按平台、品牌、年份筛选
- **用途**：查看数据同步的执行情况，确认同步是否成功

### 6.4 费用日志（/logs/fee-logs）
- **内容**：费用操作的变更记录
- **用途**：追踪费用数据的变更历史

### 6.5 用户日志（/logs/user-log）
> 仅超级管理员可访问
- **内容**：用户的操作行为记录
- **用途**：审计用户操作，追踪问题

### 6.6 新媒体 API 日志（/logs/newmedia-api-log）
- **内容**：平台 API 数据同步的调用日志
- **用途**：查看 API 调用情况，排查同步失败原因

### 6.7 采集日志（/logs/collectionlog）
- **内容**：采集脚本的执行日志
- **用途**：查看采集任务的执行情况，确认采集是否正常
