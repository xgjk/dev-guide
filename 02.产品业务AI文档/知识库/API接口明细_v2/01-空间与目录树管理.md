# 空间与目录树管理

> 包含空间发现、树形目录遍历、搜索以及文件基础属性修改接口。

## 接口列表

---

### 1.1 【基础】获取个人知识库空间ID

获取当前用户个人知识库的空间 ID（projectId），用于对应知识库目录。

> [!NOTE]
> **个人知识库特殊性说明**：
> 个人知识库本质上就是一个**拥有固定唯一归属人、无复杂权限体系的特殊 projectId**。获取到该 `projectId` 后，您可以将其传入任何带有 `projectId` 参数的**通用空间接口**（如 `2.2 saveFileByParentId` 等），从而实现向个人知识库执行高级通用操作的目的。详情可参阅《总纲》的 2.4 节。

**基本信息**

| 项目 | 说明 |
| :--- | :--- |
| 接口地址 | `/document-database/project/personal/getProjectId` |
| 请求方式 | `GET` |
| 接口负责人 | 知识库服务团队 |
| 所属模块 | 知识库服务 |
| 版本号 | v1 |
| 接口类型 | 查询 |
| 推荐调用场景 | 获取个人知识库空间标识 |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `appCode` | String | 否 | 应用编码；**不传时 OpenAPI 不会自动补默认查询参数**，由 **document-database** 结合请求头中的 `corpId` 等企业规则解析（个人知识库维度与「玄关门限 / 其他企业」的 `kz_doc` / `kz_knowledge_base` 约定一致，corpId 以线上为准） |

> [!NOTE]
> **OpenAPI 与下游**：经 `/open-api/document-database/...` 访问时，网关层对 `appCode` **不做固定默认值填充**；缺省语义由 **document-database** 在对应 Controller / Service 内处理。

**请求与行为约定**

| 项 | 说明 |
| --- | --- |
| 是否支持分页 | 否 |
| 是否支持批量 | 否 |
| 幂等性要求 | 是-天然幂等 |
| 额外字段策略 | 忽略未定义字段 |
| 返回裁剪策略 | 返回单个 Long 值 |

**请求示例**

```bash
curl -X GET 'https://{域名}/open-api/document-database/project/personal/getProjectId' \
  -H 'appKey: YOUR_API_KEY'
```

**响应示例**

```json
{
  "resultCode": 1,
  "resultMsg": null,
  "data": 2009488364113997826
}
```

**数据流向**

- 返回的 `projectId` 用于 **1.4 getLevel1Folders** / **1.10 searchFile** / **2.3 saveFileByPath** 的 `projectId` 入参。

### 1.2 【基础】获取该账号有权限访问的空间列表 [新增]

拉取当前账号有权访问（含只读）的所有空间列表，用于 Agent 进行前置的空间发现与导航。

**基本信息**

| 项目 | 说明 |
| :--- | :--- |
| 接口地址 | `/document-database/project/list` |
| 请求方式 | `GET` |
| 接口负责人 | 知识库服务团队 |
| 所属模块 | 知识库服务 |
| 版本号 | v1 |
| 接口类型 | 查询 |
| 推荐调用场景 | Agent 发现可用空间、浏览空间列表 |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `appCode` | String | 否 | 应用编码；**不传时 OpenAPI 不补默认值**，由 **document-database** 按当前企业解析：玄关门限内企业为 `kz_doc`，其余为 `kz_knowledge_base`（corpId 门限以线上为准） |
| `nameKey` | String | 否 | 空间名称模糊搜索关键词。**⚠️ 中文必须 URL 编码（UTF-8）** |
| `bizCode` | String | 否 | 业务线编码过滤（如 `pmo`） |

**请求与行为约定**

| 项 | 说明 |
| --- | --- |
| 是否支持分页 | 否 |
| 是否支持批量 | 否 |
| 幂等性要求 | 是-天然幂等（只读查询） |
| 额外字段策略 | 忽略未定义字段 |
| 返回裁剪策略 | 支持 `nameKey`/`bizCode` 过滤 |

> [!NOTE]
> 历史上部分文档写「默认 `kz_doc`」来自旧版 **OpenAPI** 的 `defaultValue`；当前约定为 **缺省由 document-database 按企业解析**，与 **1.1** 中 OpenAPI 不注入 `appCode` 默认值的说明一致。

**请求示例**

```bash

  -H 'appKey: YOUR_API_KEY'
```

**响应参数**

`data` 类型为 `List<ProjectVO>`，字段详见 **[6.1.4 ProjectVO](#614-projectvo)**。

**响应示例**

```json
{

  "resultMsg": null,
  "data": [
    {
      "id": 2025001,
      "name": "AI 研发中心",
      "remark": "存放 AI 相关的技术蓝图",
```

**数据流向**

- 返回的 `id`（即 `projectId`）用于 **1.4 getLevel1Folders** / **1.5 getChildFiles** / **1.10 searchFile** / **2.2 saveFileByParentId** / **2.3 saveFileByPath** 的 `projectId` 入参。

**基本信息**

| 项目 | 说明 |
| :--- | :--- |
| 接口地址 | `/document-database/file/getLevel1Folders` |
| 请求方式 | `GET` |
| 接口负责人 | 知识库服务团队 |
| 所属模块 | 知识库服务 |
| 版本号 | v1 |
| 接口类型 | 查询 |
| 推荐调用场景 | Agent 浏览空间根目录全景 |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `projectId` | Long | 是 | 项目/空间 id（来自 1.1 或 1.2 返回的 id） |
| `order` | Integer | 否 | 排序规则：1(更新倒序), 2(更新顺序), 5(名字倒序), 6(名字顺序) |
| `permissionQuery` | String | 否 | 权限查询条件 |

**请求与行为约定**

| 项 | 说明 |
| --- | --- |
| 是否支持分页 | 否 |
| 是否支持批量 | 否 |

**请求示例**


### 1.5 【检索】根据父ID获取下级目录及文件列表

根据父文件夹 ID 获取其下级目录及文件列表，支持排序、标签、类型等筛选。

**基本信息**

| 项目 | 说明 |
| :--- | :--- |
| 接口地址 | `/document-database/file/getChildFiles` |
| 请求方式 | `GET` |
| 接口负责人 | 知识库服务团队 |
| 所属模块 | 知识库服务 |
| 版本号 | v1 |
| 接口类型 | 查询 |
| 推荐调用场景 | Agent 遍历目录树、文件浏览 |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `parentId` | Long | 是 | 要展开的父文件夹id(根目录传0或查出的项目级id) |
| `type` | Integer | 否 | 过滤资源类型(空为所有，1只查文件夹，2只查文件) |
| `label` | String | 否 | 标签:空或者[原始]都是全部,初始化,待处理,放行 |
| `order` | Integer | 否 | 排序规则：1倒序更新 2顺序更新 3倒序创建 4顺序创建 5倒序名字 6顺序名字(AI常用1或3) |
| `excludeFileTypes` | String | 否 | 需要排除的文件业务分类，多个用逗号分隔(例如: work_report,work_plan,huiji,ai-report) |
| `excludeFolderNames` | String | 否 | 需要排除的文件夹名称，多个用逗号分隔(例如: 临时文件,测试文件夹) |
| `returnFileDesc` | Boolean | 否 | 强制带回文件描述摘要(建议AI传true) |

**响应参数**

`data` 类型为 `List<FileVO>`，字段详见 **[6.1.1 FileVO](#611-filevo)**。

**请求与行为约定**

| 项 | 说明 |
| --- | --- |
| 是否支持分页 | 否 |
| 是否支持批量 | 否 |
| 幂等性要求 | 是-天然幂等（只读查询） |
| 额外字段策略 | 忽略未定义字段 |
| 返回裁剪策略 | 支持 `type`/`excludeFileTypes`/`excludeFolderNames`/`returnFileDesc` 过滤 |

**请求示例**

```bash
curl -X GET 'https://{域名}/open-api/document-database/file/getChildFiles?parentId=1000' \
  -H 'appKey: YOUR_API_KEY'
```

**响应示例**

```json
{
  "resultCode": 1,
  "resultMsg": null,
  "data": [
    {"id": 1001, "name": "技术方案", "type": 1, "parentId": 0, "hasChild": true},
    {"id": 1002, "name": "需求文档.pdf", "type": 2, "parentId": 0, "suffix": "pdf", "size": 204800, "hasChild": false}
  ]
}
```

**数据流向**

- 返回的 `id`（文件夹类型）可作为本接口 `parentId` 继续下钻；`id`（文件类型）用于 **4.1 getDownloadInfo** / **4.2 getFileContent** / **3.2 getFullFileContent** 的入参。

### 1.6 【检索】子树扁平列举（同步/冷启动专用）[新增]

在指定根目录（`rootFileId`）下，**扁平分页**列举全部后代文件的元数据（默认仅列 `.md`），用于知识库冷启动与目录对账，减少递归 `getChildFiles` 的请求次数。

**基本信息**

| 项目 | 说明 |
| :--- | :--- |
| 接口地址 | `/document-database/file/listDescendantFiles` |
| 请求方式 | `GET` |
| 接口负责人 | 知识库服务团队 |
| 所属模块 | 知识库服务 |
| 版本号 | v1 |
| 接口类型 | 查询 |
| 推荐调用场景 | 冷启动全量拉取、目录对账、子树扫描 |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `rootFileId` | Long | 是 | 根目录 `fileId` |
| `projectId` | Long | 否 | 空间 id（建议传入，便于权限/隔离校验） |
| `suffix` | String | 否 | 文件后缀过滤，默认 `md` |
| `cursor` | String | 否 | 翻页游标（上一页返回的 `nextCursor`） |
| `limit` | Integer | 否 | 单页数量，默认 500（建议不要过大） |
| `includePath` | Boolean | 否 | 是否返回 `relativePath`，默认 false（开启会有额外路径计算成本） |

**请求与行为约定**

| 项 | 说明 |
| --- | --- |
| 是否支持分页 | 是（cursor） |
| 是否支持批量 | 是（列表返回） |
| 幂等性要求 | 是-天然幂等（只读查询） |
| 额外字段策略 | 忽略未定义字段 |
| 返回裁剪策略 | 仅返回元数据，不含正文 |

**请求示例**

```bash
curl -X GET 'https://{域名}/open-api/document-database/file/listDescendantFiles?projectId=2025001&rootFileId=10086&suffix=md&limit=500&includePath=false' \
  -H 'appKey: YOUR_API_KEY'
```

**响应示例**

```json
{
  "resultCode": 1,
  "resultMsg": null,
  "data": {
    "files": [
      {
        "fileId": 30001,
        "parentId": 10086,
        "name": "README.md",
        "updateTime": 1714972800000,
        "size": 1024,
        "relativePath": "AI生成/调研摘要/README.md"
      }
    ],
    "nextCursor": "MTcxNDk3MjgwMDAwMCwzMDAwMQ"
  }
}
```

### 1.7 【检索】增量变更列表（同步/轮询专用）[新增]

用于知识库双向同步：按 `since`（毫秒时间戳）拉取变更，并可用 `cursor` 翻页直到拉完。事件类型包含 `upsert`（新增/更新）与 `delete`（回收站/隐藏等逻辑删除）。

**基本信息**

| 项目 | 说明 |
| :--- | :--- |
| 接口地址 | `/document-database/file/listChanges` |
| 请求方式 | `GET` |
| 接口负责人 | 知识库服务团队 |
| 所属模块 | 知识库服务 |
| 版本号 | v1 |
| 接口类型 | 查询 |
| 推荐调用场景 | 客户端增量同步、对账轮询、变更追踪 |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `projectId` | Long | 否 | 空间 ID；不传时服务端会**默认使用个人知识库**（行为等价于先调用 **1.1 getProjectId** 获取个人空间 `projectId`）。**前置条件**：需能从请求头获取 `employeeId/corpId`；若 `corpId` 缺失或该企业/场景不开放个人空间，则接口会返回错误（不会退化为跨空间查询）。 |
| `rootFileId` | Long | 否 | 限定在某个子树内同步 |
| `since` | Long | 否 | 首次/重建同步水位（毫秒时间戳） |
| `cursor` | String | 否 | 翻页游标（存在时优先于 since） |
| `limit` | Integer | 否 | 单页数量，默认 200 |

**请求与行为约定**

| 项 | 说明 |
| --- | --- |
| 是否支持分页 | 是（cursor） |
| 是否支持批量 | 是（列表返回） |
| 幂等性要求 | 是-天然幂等（只读查询） |
| 额外字段策略 | 忽略未定义字段 |
| 返回裁剪策略 | 仅返回变更元信息，不含正文 |

**请求示例**

```bash
curl -X GET 'https://{域名}/open-api/document-database/file/listChanges?projectId=2025001&rootFileId=10086&since=1714972800000&limit=200' \
  -H 'appKey: YOUR_API_KEY'
```

**响应示例**

```json
{
  "resultCode": 1,
  "resultMsg": null,
  "data": {
    "items": [
      {
        "fileId": 30001,
        "parentId": 10086,
        "type": 2,
        "name": "README.md",
        "updateTime": 1714972800000,
        "event": "upsert"
      },
      {
        "fileId": 30002,
        "parentId": 10086,
        "type": 2,
        "name": "old.md",
        "updateTime": 1714972810000,
        "event": "delete"
      }
    ],
    "nextCursor": "MTcxNDk3MjgxMDAwMCwzMDAwMg",
    "serverTime": 1714972812345
  }
}
```

### 1.13 【写操作】更新文件属性（重命名/移动/覆盖）

【Agent 提示】支持重命名和跨目录移动。
> [!IMPORTANT]
> **移动逻辑强调**：如果是移动操作且目标是一个文件夹，系统会**同时将该文件夹下所有的 children（子项）都带着一起移动**。

**冲突处理策略（重要）**:
同名冲突时，系统支持三选一策略：
1. **强制覆盖（⚠️ 危险，慎用！）**：传 `cover=true`。**注意覆盖的具体含义**：例如将 A.md 重命名或移动为 B.md，若同目录下已存在 B.md，系统会**先彻底删除已存在的 B.md 文件/文件夹**，然后再执行重命名或移动。这会导致原有 B.md 数据直接丢失。
2. **自动追加后缀**：传 `autoRename=true`（如 `文件名(1).pdf`）。
3. **严格报错**：如果 `autoRename` 和 `cover` 都不传，碰到重名冲突会直接报错。

请 Agent 依据人类用户的意向选择合适的策略参数。

**基本信息**

| 项目 | 说明 |
| :--- | :--- |
| 接口地址 | `/document-database/file/updateFileProperty` |
| 请求方式 | `POST` |
| Content-Type | `application/json` |
| 接口负责人 | 知识库服务团队 |
| 所属模块 | 知识库服务 |
| 版本号 | v1 |
| 接口类型 | 写入 |
| 推荐调用场景 | Agent 重命名或移动文件 |

**请求参数**

请求体为 `FileModifyParam`，主要字段如下：

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `fileId` | Long | 是 | 文件ID |
| `newName` | String | 否 | 新文件名（为空即可仅做移动） |
| `targetParentId` | Long | 否 | 目标父目录ID（为空即可仅做重命名） |
| `cover` | Boolean | 否 | 同名冲突时是否强制覆盖。**慎用！会先删除已存在的同名文件/文件夹**（cover 与 autoRename 互斥，cover 优先级更高） |
| `autoRename` | Boolean | 否 | 同名冲突时是否自动追加数字后缀重命名（cover=true 时本字段被忽略） |

**请求与行为约定**

| 项 | 说明 |
| --- | --- |
| 是否支持分页 | 否 |
| 是否支持批量 | 否 |
| 幂等性要求 | 否-重复移动可能因覆盖策略产生不同结果 |
| 额外字段策略 | 忽略未定义字段 |
| 返回裁剪策略 | 不适用 |

**请求示例**

```bash
curl -X POST 'https://{域名}/open-api/document-database/file/updateFileProperty' \
  -H 'appKey: YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{"fileId":30001,"newName":"新版报告.pdf","autoRename":true}'
```

**响应参数**

`data` 类型为 `Boolean`，表示操作成功与否。

**响应示例**

```json
{
  "resultCode": 1,
  "resultMsg": null,
  "data": true
}
```

**数据流向**

- 无下游接口依赖。操作后文件 `id` 不变，可通过 **1.5 getChildFiles** 重新浏览变更后的目录结构。
