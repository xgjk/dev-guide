# UI终端预览与阅读

> 提供给前端和客户端使用的在线预览链接和流式分页读取能力。

## 接口列表

---

### 4.1 【获取】获取文件下载与在线预览凭据

根据文件 ID 获取下载 URL 或在线预览所需凭据信息。

**基本信息**

| 项目 | 说明 |
| :--- | :--- |
| 接口地址 | `/document-database/file/getDownloadInfo` |
| 请求方式 | `GET` |
| 接口负责人 | 知识库服务团队 |
| 所属模块 | 知识库服务 |
| 版本号 | v1 |
| 接口类型 | 查询 |
| 推荐调用场景 | 文件下载与在线预览 |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `fileId` | Long | 是 | 文件 id |
| `path` | String | 否 | 基于文件id的相对路径 |
| `fileName` | String | 否 | 文件名 |
| `forceDownload` | Boolean | 否 | 是否强制下载（true 下载，false 在线预览） |
| `seeOriginal` | Boolean | 否 | 预览是否查看原文 |
| `source` | String | 否 | 来源 |
| `versionNumber` | Integer | 否 | 版本号 |
| `bypassRisk` | Boolean | 否 | 是否绕过风险检查 |

**响应参数**

`data` 类型为 `DownloadInfoVO`，字段详见 **[6.4.1 DownloadInfoVO](#641-downloadinfovo)**。

**请求与行为约定**

| 项 | 说明 |
| --- | --- |
| 是否支持分页 | 否 |
| 是否支持批量 | 否 |
| 幂等性要求 | 是-天然幂等 |
| 额外字段策略 | 忽略未定义字段 |
| 返回裁剪策略 | 返回全部字段 |

**请求示例**

```bash
curl -X GET 'https://{域名}/open-api/document-database/file/getDownloadInfo?fileId=20001' \
  -H 'appKey: YOUR_API_KEY'
```

**响应示例**

```json
{
  "resultCode": 1,
  "resultMsg": null,
  "data": {
    "downloadUrl": "https://oss.example.com/signed-urlxxx",
    "previewUrl": "https://cwork-web-test.xgjktech.com.cn/p/1c6e69e174e34001",
    "fileId": 20001,
    "openWith": 2,
    "lazyLoad": true,
    "fileName": "需求文档.pdf",
    "suffix": "pdf",
    "size": 204800
  }
}
```

**数据流向**

- `previewUrl`：在线预览短链接（`forceDownload=false` 时填充），**Agent 应将此链接提供给用户点击预览**
- `downloadUrl`：OSS 签名直链，用于文件下载（`forceDownload=true` 时使用）
- `openWith`：前端渲染方式决策，供需要精细控制渲染的场景使用

### 4.2 【获取】UI阅读模式：根据文件ID和页码分页获取内容

主要供 UI 界面对文件进行分段、分页的分批流式展示。
**对纯文本类文件无分页概念，会忽略 `pageNumber` 直接返回全量正文**——详见下文"行为契约"。

**基本信息**

| 项目 | 说明 |
| :--- | :--- |
| 接口地址 | `/document-database/file/getFileContent` |
| 请求方式 | `GET` |
| 接口负责人 | 知识库服务团队 |
| 所属模块 | 知识库服务 |
| 版本号 | v2 |
| 接口类型 | 查询 |
| 推荐调用场景 | UI 分页展示文件内容；对纯文本类文件可作为"全量正文"获取入口（与 3.2 等价） |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `fileId` | Long | 是 | 文件 id |
| `pageNumber` | Integer | 否 | 页码，从 1 开始；**对"无分页类型"文件该参数被忽略** |

**响应参数**

`data` 类型为 `String`：
- 有分页类型 → 该页的排版文本；
- 无分页类型 → 全量正文。

**行为契约（v2 起）**

按"文件后缀"决定是否按页返回：

| 文件类型 | 分页能力 | `pageNumber` 含义 | 返回值 |
| :--- | :--- | :--- | :--- |
| `pdf` / `pptx` / `ppt` / `docx` / `doc` 等 | **有分页** | 必填，从 1 开始 | **该页**的排版文本 |
| `md` / `txt` / `html` / `htm` / `xml` / `notex` / `xls` / `xlsx` | **无分页** | **被忽略** | **全量正文**（与 3.2 `getFullFileContent` 同源、内容一致） |
| 其他/未知后缀 | 兜底按"有分页"处理 | 视为页码 | 远端按页切片返回；若该格式不支持文本提取，返回空字符串 |

> 设计原因：上述"无分页类型"在 file-convert 服务里没有"按页切片"的索引，
> 历史实现直接按页码取必然得到空字符串；v2 起后端按 suffix 分流，对纯文本类一律走全量抽取，
> 避免对调用方产生"成功但 `data: ""`"的歧义返回。

**异常返回**

| 场景 | 返回 |
| :--- | :--- |
| `fileId` 不存在 / 已删除 | `data: null` |
| `resourceId` 未生成 | `data: null` |
| 远端 file-convert 抽取异常 | `data: null`（所有降级策略均失败） |

**降级策略（v2 起）**

| 文件类型 | 降级链 |
| :--- | :--- |
| 无分页类型（md/txt/html 等） | file-convert 全量抽取 → 直接下载原始文件内容 → FileRag 表 → null |
| 有分页类型（pdf/docx 等） | 指定页 → 第1页 → 全量抽取 → FileRag 表 → null |

**请求与行为约定**

| 项 | 说明 |
| --- | --- |
| 是否支持分页 | 仅"有分页类型"支持；"无分页类型"忽略 `pageNumber` |
| 是否支持批量 | 否 |
| 幂等性要求 | 是-天然幂等 |
| 额外字段策略 | 忽略未定义字段 |
| 返回裁剪策略 | 有分页类型按页返回单页；无分页类型返回全量 |

**请求示例**

```bash
# 1) 有分页类型（PDF）：取第 1 页
curl -X GET 'https://{域名}/open-api/document-database/file/getFileContent?fileId=30001&pageNumber=1' \
  -H 'appKey: YOUR_API_KEY'

# 2) 无分页类型（md/txt 等）：pageNumber 可省略，返回全量正文
curl -X GET 'https://{域名}/open-api/document-database/file/getFileContent?fileId=40002' \
  -H 'appKey: YOUR_API_KEY'
```

**响应示例**

```json
{
  "resultCode": 1,
  "resultMsg": null,
  "data": "# 第一章 概述\n\n本文档描述了..."
}
```

**与 3.2 `getFullFileContent` 的选择建议**

- 想要"按页 / 流式 / UI 阅读体验" → 用 4.2，并对有分页类型传入 `pageNumber`。
- 想要"一次拿全文喂给 AI / 拼接 prompt" → 用 3.2。
- 对**纯文本类**文件（md/txt/html 等），4.2 与 3.2 返回内容**等价**；首选 3.2 表达"我要全量"语义更清晰。

**数据流向**

- 无下游接口依赖。

## 数据结构

### 6.4.1 DownloadInfoVO
下载/在线预览凭据信息。

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `downloadUrl` | String | 下载直链（OSS 签名 URL），用于文件下载 |
| `previewUrl` | String | **在线预览短链接**（`forceDownload=false` 时填充），可直接给用户点击预览；`forceDownload=true` 时为 `null`。**Agent 应优先使用此字段给用户展示预览链接** |
| `thumbnailUrl` | String | 图片缩略图 url |
| `fileName` | String | 文件名 |
| `versionNumber` | Integer | 版本号 |
| `fileId` | Long | 文件 id |
| `suffix` | String | 文件后缀 |
| `size` | Long | 文件大小（字节） |
| `fileType` | String | 关联的业务类型。枚举：doc(富文本), file(普通文件), work_report(工作汇报), work_plan(工作任务), url(链接), huiji(慧记), ai-report(AI情报) |
| `fileContent` | String | 文件内容 |
| `resourceId` | Long | 资源 id |
| `openWith` | Integer | 打开方式：0 默认或下载 / 1 wps / 2 pdf / 3 畅写 / 4 html / 5 工作协同 / 6 pdf-v5 |
| `lazyLoad` | Boolean | 是否按需加载（目前用于 pdf.js 加载 pdf 文件） |
| `lastVersion` | Boolean | 是否最新版本 |
| `projectId` | Long | 文件空间 Id |
